<objective>
Make every Codex Responses request — whether built via a mid-session switch, an initial session start,
or a mid-turn config reconstruction (auth refresh/retry) — carry a `reasoning.effort` value that is
always either a member of the target model's catalog-advertised supported list or omitted entirely,
and align `reasoning.summary` omission with real catalog data (a new `default_reasoning_summary_none`
field on `ModelInfo`, populated for GPT-5.6), with zero change to Grok/xAI wire behavior. This lands the
single choke point (per D-locked decision: `conversation.rs` `From<&ConversationRequest> for
rs::CreateResponse`, `conversation.rs:2159`) plus the catalog-supported-levels and catalog-summary-default
data threading it needs, and the pure clamp function that ports official Codex semantics (`codex-rs`
`core/src/session/turn_context.rs:243-263`).

Purpose: this is the load-bearing correctness layer for phase goal criteria 3 (wire sends
`reasoning.effort` for supported levels only) and criterion 5 (summary omit aligned with catalog `none`
default). Cross-AI review (`11-REVIEWS.md`, HIGH finding 1) found the original design left initial
(non-switch) Codex sessions and mid-turn config reconstruction on a separate production path that never
received the new catalog fields — this revision adds those two call sites explicitly so "every Codex
Responses request" is actually true, not just "every request built via a mid-session switch". It is
intentionally scoped to wire-shape correctness — it does not yet make the mid-session switch itself
sticky/clamp-aware (that is `11-02-PLAN.md`, which depends on the `clamp_reasoning_effort` helper and the
`reasoning_effort_supported`/`reasoning_summary_omit` fields this plan introduces).
Output: a new pure `clamp_reasoning_effort` function; two new catalog-derived fields
(`reasoning_effort_supported`, `reasoning_summary_omit`) threaded through `SamplerConfig` ->
`SamplingConfig` -> `ConversationRequest` AND through every production site that constructs those types
(switch path, initial-session spawn path, mid-turn reconstruction path); a new `ModelInfo` catalog field
(`default_reasoning_summary_none`) populated for GPT-5.6 Sol/Terra/Luna; updated choke-point emission
logic for both `reasoning.effort` and `reasoning.summary`; and wire/unit/integration tests proving the
whole path, including the initial-session path (review finding M2).
</objective>
