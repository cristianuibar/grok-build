<objective>
Make every Codex Responses request carry a `reasoning.effort` value that is always either a member of
the target model's catalog-advertised supported list or omitted entirely, and align `reasoning.summary`
omission with the catalog default — with zero change to Grok/xAI wire behavior. This lands the single
choke point (per D-locked decision: `conversation.rs` `From<&ConversationRequest> for rs::CreateResponse`,
`conversation.rs:2159`) plus the catalog-supported-levels data threading it needs, and the pure clamp
function that ports official Codex semantics (`codex-rs` `core/src/session/turn_context.rs:243-263`).

Purpose: this is the load-bearing correctness layer for phase goal criteria 3 (wire sends
`reasoning.effort` for supported levels only) and criterion 5 (summary omit aligned with catalog `none`
default). It is intentionally scoped to wire-shape correctness — it does not yet make the mid-session
switch itself sticky/clamp-aware (that is `11-02-PLAN.md`, which depends on the `clamp_reasoning_effort`
helper and the `reasoning_effort_supported` field this plan introduces).
Output: a new pure `clamp_reasoning_effort` function, a new `reasoning_effort_supported` field threaded
through `SamplerConfig` -> `SamplingConfig` -> `ConversationRequest`, updated choke-point emission logic,
and wire/unit tests proving the whole path.
</objective>
