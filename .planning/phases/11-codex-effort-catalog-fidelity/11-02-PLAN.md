<objective>
Make the mid-session model switch itself sticky and clamp-aware, and surface the clamp to the user. The
session's chosen reasoning effort now survives every switch (server-side, keyed off the existing
`SessionHandle.reasoning_effort` field) instead of resetting to the target model's catalog default on
every switch (today's bug, `.planning/phases/11-codex-effort-catalog-fidelity/11-RESEARCH.md` §B5 finding
2). The switch handler never silently drops an unsupported override (today's bug, §B5 finding 3/4) — it
always stores the raw preference and lets `11-01-PLAN.md`'s choke point clamp it per-request. A one-line,
non-blocking TUI notice (locked copy from `11-UI-SPEC.md`) tells the user when a switch produced an
effective effort different from what they had.

Purpose: closes phase goal criteria 2 (soft-clamp, never hard-fail) and criterion 4 (UAT notes no longer
list effort as a blocker). Depends on `11-01-PLAN.md`'s `clamp_reasoning_effort` helper and
`reasoning_effort_supported` field — this plan calls the same shared function a second time (mirrored,
display-only) at switch time, per CONTEXT's "shared function all callers route through."
Output: sticky per-session effort preference, clamp-aware switch handler, ACP response meta carrying the
resolved effort + clamped flag, TUI notice, behavior + TUI test tiers, and UAT-notes closure.
</objective>
