## Final-gate filter failure policy (HARD — C1-M7)

When a required filter is missing (discover n=0) or fails at final gate:

1. **Stop** — do not mark automated half green.
2. **Diagnose** target/filter mismatch (wrong --lib vs --test; rename from Plan 04 fix).
3. **Restore** the original behavioral test coverage or update PHASE-GATE to the real greened prior-phase name — preserve required filter names where possible.
4. **Forbidden as default:** adding a thin discoverable wrapper solely to make n≥1 while the original behavioral assertion is gone or weakened.
5. **Exception:** a replacement test is allowed only with a reviewer-visible explanation in 09-VERIFICATION.md / SUMMARY stating what was lost, what was restored, and why the original could not be kept. Prefer restoring original over inventing wrapper.

## Secrets GREEN gate (HARD — C1-M8)

Before claiming GREEN:

1. Content scan: no JWT-shaped strings / private key blocks in phase artifacts.
2. Path guard: `git ls-files` must not include `auth.json` or credential token filenames under phase artifacts/repo tracking.
3. Phase-diff scan: inspect `git diff` for this phase's commits for auth.json contents, opaque OAuth token dumps, and credential-like values — not only two secret regex shapes.
4. Optional: `git diff --check` cleanliness for phase files.
