## Plan user story

**As a** bum daily-driver user, **I want to** complete GPT-5.6 turns and Grok↔GPT-5.6 switches reliably, **so that** I can keep coding in one CLI without a restart.

<objective>
Repair the Responses SSE reconstruction path so visible successful text cannot become an empty completed response and trigger a post-success retry storm.

Purpose: make OPS-04 completion semantics truthful without weakening retries for genuinely malformed empty responses.
Output: a narrow text fallback in the stream transformer plus unit and actor-level retry regressions after Plan 01 establishes `test_actor.rs` configuration compatibility.
</objective>
