## Plan user story

**As a** bum daily-driver user, **I want to** complete GPT-5.6 turns and Grok↔GPT-5.6 switches reliably, **so that** I can keep coding in one CLI without a restart.

<objective>
Create the disabled-by-default typed contract that lets a trusted ChatGPT/Codex caller request Codex Responses wire behavior without making endpoint or provider decisions in the sampler.

Purpose: make OPS-04 request construction explicit and prove the generic conversion remains safe for non-Codex callers.
Output: a profile field propagated through sampler defaults, bounded sampler-crate literal compatibility updates, profile-aware request shaping, and profile-on/profile-off serialization tests.
</objective>

## Locked decisions honored

- This plan introduces a semantic trusted-Codex profile only; shell code remains the source of endpoint/OAuth trust, and sampler code must not infer trust from a URL.
- Plan 04 is the sole live activation owner: this plan deliberately supplies typed plumbing and regression coverage before the trusted first-party Codex OAuth gate elects the profile.
- The profile makes `store: false`, streaming, encrypted-reasoning include, full-history HTTP continuity, and omitted summary explicit for Codex while retaining generic conversion behavior elsewhere.
- Bum session identity remains primary. Optional headers are request metadata owned by the later trusted-shell plan, not account identity or persisted third-party identity.
- Phase 11 owns reasoning capability/effort policy; Phase 12 owns WebSocket, Codex tool-name/freeform parity, and broader attribution work.
