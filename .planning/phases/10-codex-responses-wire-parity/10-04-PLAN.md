**As a** bum daily-driver user, **I want to** complete GPT-5.6 turns and Grok↔GPT-5.6 switches reliably, **so that** I can keep coding in one CLI without a restart.

<objective>
Activate the Phase 10 trusted wire profile and request identity only at the existing first-party ChatGPT/Codex OAuth boundary, and centralize encrypted-content mismatch classification.

Purpose: complete trusted Codex interoperability without leaking identity metadata or recovery behavior to xAI, BYOK, or custom endpoints.
Output: shell-owned trusted profile/header construction, isolation tests, and a single 400-gated encrypted-content predicate used by friendly recovery.
</objective>

## Locked decisions honored

- Reuse `is_first_party_codex_url` plus Codex `SessionToken` OAuth as the exclusive trust gate; do not add URL guessing to sampler code.
- Use bum's existing session identity as the stable local source for optional UUID metadata. These headers are request metadata, not a replacement identity or third-party state store.
- Send `originator: bum`, never the upstream CLI value, and do not add telemetry, attestation, installation, or beta headers.
- Keep `ChatGPT-Account-ID` under the current trusted-and-known behavior. Phase 11/12 deferrals remain unchanged.
