## Evidence Transition Protocol

Every implementation task has exactly one row below, even when rows share a command.

1. Leave **File Exists**, **Evidence**, and **Status** at their initial pending values until the owning task has produced every listed file.
2. After the task commit, run its **Task Smoke (<30s)** command. A zero exit records fast feedback but does not by itself turn the row GREEN.
3. At the stated wave/phase boundary, run the row's **Boundary Evidence** command. Only exit code 0 permits changing **File Exists** to `✅`, **Evidence** to `PASS YYYY-MM-DD: <exact command or gate result label>`, and **Status** to `✅ green`.
4. Any nonzero run becomes `❌ red`; any nondeterministic rerun becomes `⚠️ flaky`. Preserve the command output or gate result label in **Evidence**. Never mark a row green from file presence, a prior phase, or a manually flipped boolean.
5. For `12-07-03`, run `--validation-pre-final` with the first 17 rows GREEN, the final row pending, and both flags false; run `--validation-finalize-row` to change only the final row; run `--validation-post-row` to prove all 18 rows GREEN with zero pending/red/flaky while both flags remain false; run the complete phase gate; then run `--validation-set-wave0`, `--validation-set-nyquist`, and the read-only `--validation-final` audit in that order.
