## Shared production cores + test hooks (review HIGH — locked)

Integration tests cannot call `pub(crate)` / `pub(super)`. Therefore extract **public** pure/shared functions that production methods **must** call (thin wrappers), and test those — this is the real prepare/reconstruct logic path, not a parallel toy.

Required public surface (names may vary slightly; behavior fixed):

1. `pub fn session_key_for_model_provider(provider, xai, codex) -> Option<&str>`
2. `pub fn should_attach_xai_auth_manager_bearer_resolver(provider: ModelProvider, session_token_gate_active: bool) -> bool` — true only for `Xai && gate`
3. `pub fn select_provider_access_token(store: &AuthStore) -> Option<GrokAuth>` (or returns key String) — **selection rules below**
4. `pub fn prepare_sampling_credentials(model, xai_session, codex_session, ...) -> ResolvedCredentials` — body used by prepare + ModelsManager
5. `pub fn next_sample_after_model_switch(prepare_result fields → chat_state SamplingConfig+Credentials → reconstruct attach decision) -> NextSampleRoute` — pure orchestration used by tests to simulate prepare → SetSessionModel fields → reconstruct policy **without full ACP session**
6. Extend `ModelAuthFacts` with `pub provider: ModelProvider` filled from catalog in `resolve_model_auth_facts` / `with_resolved_model`

Production must call these; tests call these. Do **not** only test helpers that prepare forgets to call.

Optional `#[doc(hidden)]` module ok if re-exported for integration tests.

## Codex token selection (review HIGH — locked)

**Forbidden:** “first entry in BTreeMap” / `values().next()`.

**Required** `select_provider_access_token` (mirror `lookup_auth` quality):

1. Consider each scope in the provider’s `AuthStore`
2. Skip `AuthMode::WebLogin` (same as `lookup_auth`)
3. Skip blank/whitespace keys
4. Prefer non-expired tokens (use existing `is_expired` / buffer helpers when available at this layer)
5. Prefer `AuthMode::Oidc` (session OAuth) over `ApiKey` over `External`
6. If multiple remain, prefer the scope key matching any future Phase 5 constant when present; else stable preference: Oidc winners sorted by scope string for determinism (document as fixture-stable, not “first arbitrary”)
7. Return None if nothing valid

Phase 5 will pin the real ChatGPT OAuth scope constant; selection function must accept that without redesign.

## Provider recovery (review HIGH — locked)

- **Do not** decide bearer attach from URL host heuristics as primary.
- Use `ModelAuthFacts.provider` from catalog lookup of `cfg.model`.
- If catalog lookup fails (`byok` Unknown / model empty): **fail closed** — do **not** attach xAI AuthManagerBearerResolver (treat like BYOK/unknown: no xAI live resolver).
- URL may still drive `inject_url_derived_headers` and SessionTokenAuthGate host checks as today, but **credential ownership / resolver attach uses provider field**.

## Snapshot Codex limitation (review MEDIUM — explicit)

Phase 4: Codex samples use Credentials.api_key snapshot set at prepare/switch (from disk slot read). Full live multi-principal Codex AuthManager refresh is **Phase 5**. Document in code comments near reconstruct Codex branch and in SUMMARY.

## Hot-path auth.json reads (review MEDIUM)

- Prefer reading Codex slot once at prepare/switch and snapshot into chat-state Credentials.
- Avoid re-reading auth.json on every reconstruct for Codex if snapshot is present.
- If a disk read helper is used at prepare, document blocking I/O is prepare/switch only (not per-stream chunk). Caching optional; snapshot is the Phase 4 requirement.
