## Locked decisions covered

D-03 (next sample rebuild — fix builders only), D-08 (reuse sampler), D-09 (no cross-slot), D-10 (live bearer preferred where safe), D-11 (fixture inject), D-13 (no OAuth/gate UX).

**Do not** invent a new ACP switch path — `handlers/model_switch.rs` already calls prepare → SetSessionModel.

**BearerResolver policy (Pitfall 2):** Today `AuthManagerBearerResolver` always returns xAI `current_or_expired()`. If left active for Codex samples, it overwrites Codex Authorization. Required behavior:
- Attach xAI AuthManager resolver only when the active sample is **xAI** (provider or non-Codex base) **and** existing SessionTokenAuthGate would attach it.
- For **Codex**: do **not** attach xAI AuthManagerBearerResolver. Prefer Credentials.api_key snapshot set at switch/prepare (fixture or future OAuth). Optional: provider-scoped resolver that reads Codex slot only if a cheap read exists — acceptable either way if tests prove no xAI overwrite.

**ShellAuthCredentialProvider:** leave on xAI AuthManager (storage/upload). Do not make it global multi-provider.

## Public test surface for private methods

`prepare_sampling_config_for_model` is `pub(crate)`; `reconstruct_full_config` is `pub(super)`. Integration tests cannot call them. Therefore:

1. Extract/keep public pure helpers used by both prepare and reconstruct policy, e.g.:
   - `pub fn select_session_keys_for_model(provider, xai_key: Option<&str>, codex_key: Option<&str>) -> Option<&str>`
   - `pub fn should_attach_xai_auth_manager_bearer_resolver(provider: ModelProvider, use_session_token_gate: bool) -> bool` — true only for Xai + gate
2. Optional public `pub fn first_provider_access_token(auth_file: &Path, provider: &str) -> Option<String>` reading nested auth document for Codex fixture (storage.rs) — for prepare path + tests without full OAuth.
3. Prove switch via `sampling_config_for_model` + dual entries (already Plan 01/03) plus pure policy tests for bearer attach.
