## Local fail-closed (review HIGH — locked)

Today `SamplingClient::new` accepts `api_key: None` and sends unauthenticated HTTP; remote 401 becomes `SamplingError::Auth`. That is **not** Phase 4 fail-closed.

Required:

1. Before any network I/O for a live sample, if **both** `config.api_key` is None/blank **and** `config.bearer_resolver` is None (or resolver returns None at preflight if checked at new/request start), return `SamplingError::Auth` (or existing typed auth error) **locally**.
2. Test: mock server receives **zero** requests when credentials absent.
3. Route **construction** with empty key remains allowed for pure resolve/tests (D-11) — fail-closed is the **sample client / live sample** boundary, not resolve_credentials.

Placement discretion: prefer `SamplingClient::new` and/or first request entry so all call paths covered. Shell prepare may also warn with provider field (already planned).

Do **not** add Phase 6 login modal/prompt.

## On-wire Authorization gate (review HIGH — locked)

Config-field equality alone is insufficient for phase gate.

Add async integration tests in `provider_routing.rs` (or dedicated section) using the existing `MockInferenceServer` pattern from `tests/test_sampling_client.rs`:

| Case | base_url | api_key | expect Authorization | expect X-XAI-Token-Auth |
|------|----------|---------|----------------------|-------------------------|
| xAI/proxy fake | proxy or mock URL with proxy header rules as applicable | xai-fake-token | `Bearer xai-fake-token` | present only if URL classified as cli-chat-proxy |
| Codex fake | mock URL standing in for Codex (client points at mock) | codex-fake-token | `Bearer codex-fake-token` | **absent** |
| missing both | mock URL | None, no resolver | **no request** + local Auth error | n/a |

After model-switch simulation (Plan 04 next_sample helper), building a client with Codex snapshot must not send xai-fake-token.

If MockInferenceServer is not re-exportable from shell tests easily, duplicate minimal wiremock/axum mock following test_sampling_client patterns inside provider_routing or extract shared test util — keep in shell test binary, no new crates.
