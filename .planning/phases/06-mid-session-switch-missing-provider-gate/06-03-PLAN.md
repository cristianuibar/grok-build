Purpose: User logs in for the missing provider without re-picking the model — whether login is in-process or `bum login --provider codex` in another terminal.
Output: Deferred stash + auth complete retry + status-refresh path GREEN with unit tests (fixture/sim — no live OAuth required).
</objective>

## Discretion lock (Login now depth)

1. **xAI Login now:** Reuse existing mid-session login / Authenticate effect path when provider is xai (dispatch_login / Effect::Authenticate patterns in auth.rs).
2. **Codex Login now:** If no thin Codex TUI authenticate effect exists, close QuestionView, set deferred_model_switch with required_provider=codex, push system/scrollback or toast restating `bum login --provider codex`, and start **provider-status refresh wait** (below). Do **not** call xAI interactive login for a Codex-missing gate.
3. Never apply the pending model until the required provider is usable **and** shell switch gate would pass (shell still authoritative on retry).

## External CLI → deferred apply (contract)

`handle_auth_complete` alone is insufficient for CLI-primary Codex (review HIGH). Required path:

1. **Producer:** Reuse Plan 04 refresh of dual-slot usable flags (AuthStatusReport / AuthMeta providers / Effect::RefreshProviderAuthStatus). Pure booleans only — no tokens in AppView.
2. **Triggers while deferred is pending with required_provider:**
   - Successful in-process AuthComplete (existing path) → refresh then maybe apply
   - After CLI-primary Login now: schedule lightweight poll (Effect / interval / on next event-loop tick batch) while deferred remains pending — cap polls reasonably (e.g. every few seconds for a limited window, or until focus cycle) so idle TUI does not spin forever
   - Terminal / app focus regain (if a focus event already exists in the event loop; otherwise document poll as primary) → one-shot refresh
   - Optional: auth.json mtime watch if fsnotify is already process-global and cheap — prefer poll/focus if watcher is heavy
3. **Consumer:** `try_apply_deferred_model_switch_if_ready(app)`:
   - If deferred is None → no-op
   - If deferred.required_provider is Some(p) and !cache.usable(p) → keep deferred, no SwitchModel
   - If required provider usable (or required_provider is None for legacy CLI `-m` deferrals) → take deferred, emit Effect::SwitchModel with model_switch_pending
4. Unrelated successful auth for the other provider must **not** apply a codex-targeted deferred while codex still unusable.
