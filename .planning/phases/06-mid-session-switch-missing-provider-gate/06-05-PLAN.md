## Session harness contract (non-negotiable)

Do **not** treat `provider_routing::switch_changes_next_sample_route` as proof of history preservation or mid-turn non-cancel — it only composes routing transforms without MvpAgent, chat history, or in-flight turns (review finding).

**Required harness** (extend Plan 01 `tests/model_switch_gate.rs` or co-located helpers):

1. Real session path capable of `model_switch::apply` / ACP `set_session_model` with dual-slot fixture auth and mixed catalog. Patterns: `session_load_perf.rs`, `git_contention_e2e.rs`, `test_support` headless + MvpAgent.
2. **History — named observable (no substitutes):**
   - Seed **at least one completed user + assistant exchange** into the session under test (session storage `chat_history.jsonl` load via existing session load helpers, or chat-state append API available on the harness — pick **one** and name it in SUMMARY).
   - Snapshot **before** successful cross-provider apply: message/turn count **and** at least one stable identity (message id, turn index, or exact user prompt string present).
   - After apply success: assert same count **and** same identity/content still present.
   - Forbidden: empty-history "preserved" tautology; pure unit stub without session storage/chat-state; provider_routing-only.
3. **Mid-turn non-cancel — named observable (no substitutes):**
   - Use `xai_grok_test_support::MockInferenceServer::hold_agent_completions` (or equivalent held mock) so a prompt turn is **in-flight** (request received, completion not released).
   - While held, call `model_switch::apply` / set_session_model for a dual-usable free switch; assert apply succeeds.
   - Assert **no** `session/cancel` (or cancel-turn / abort equivalent on the session harness cancel path) is issued during apply — observe via harness cancel counter, notification spy, or session command log already used in `test_leader_stdio_integration` cancel tests. Name the counter/spy in SUMMARY.
   - Release hold after; turn may complete on the **new** model route or stay mid-flight per product — either is OK if cancel was not sent.
   - Forbidden: "apply source does not mention cancel" static code inspection alone; provider_routing-only.
4. **Next-turn provider:** after successful switch, assert prepared sampling / next sample credentials / base_url match the **new** provider at session/apply level (Phase 4 route fields on prepared config).
5. All free-switch / history / mid-turn tests use unique `p6_` prefixes.

`provider_routing.rs` may still host pure routing regression `switch_changes_next_sample_route` (already green) — optional dual_login pure helpers only if they do not claim session history proofs.
