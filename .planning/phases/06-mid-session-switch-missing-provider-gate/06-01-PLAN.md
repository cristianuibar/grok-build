## Phase Goal

**As a** bum user mid-session, **I want to** switch models freely and get a clear fail-closed login path when a provider is missing, **so that** I never depend on silent mid-turn 401 as the primary UX and never restart the CLI to change models.

<objective>
Ship the authoritative shell missing-provider gate and typed ACP error (per D-01, D-02, D-07, UI-SPEC). After this plan, `session/set_model` / `model_switch::apply` blocks unusable-provider targets with `MODEL_SWITCH_MISSING_PROVIDER` before any sampling apply, with tests that exercise `apply` side-effect absence (review cycle 1).

Purpose: Single authority so TUI, `/model`, settings live switch, and ACP clients cannot diverge (MOD-06 core).
Output: Error type + usable helper + gate in `apply` + automated shell proofs that exercise the real apply path (fixture tokens only).
</objective>
