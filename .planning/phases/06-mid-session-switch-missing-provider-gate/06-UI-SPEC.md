# Phase 6 — UI Design Contract

> Visual and interaction contract for **mid-session model switch** and the
> **missing-provider gate** surfaces.
> Product is a **ratatui terminal TUI + ACP + CLI**, not a web app. Tokens map
> to `Theme` fields and character-cell layout (TUI) or plain stderr/stdout /
> structured ACP errors (headless), not CSS/shadcn.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (not applicable — TUI/CLI/ACP, not web) |
| Preset | not applicable |
| Component library | ratatui 0.29 + existing picker / slash dropdown / QuestionView / scrollback system blocks / toast |

## Typography

Terminal monospaced — “size” is **role weight + Theme color**, not px. Exactly **3 roles**, **2 weights**.

| Role | Size (cells) | Weight | Line Height | Theme / style |
|------|--------------|--------|-------------|---------------|
| Body (primary row label, modal question, scrollback system line) | 1 row | regular (400) / **bold when selected row** | 1.0 | `text_primary`; selected + `Modifier::BOLD` |
| Label (provider names, option labels, field keys) | 1 row | regular (400); **bold only on selected picker row** | 1.0 | Provider human labels plain (`xAI`, `Codex`) |
| Secondary (auth badge, option description, CLI fallback, dim hints) | 1+ rows | regular (400) | 1.0 | `gray` / `text_secondary` |
| Display | — | — | — | **not used** (no new large titles) |

**Rules:**

- Max **2** font weights: regular + bold (selected picker row only)
- Gate modal question = Body regular; options follow existing QuestionView styles
- Auth badge never bold; never uses accent colors
- Model name in success confirmation may appear in prompt chrome via existing `accent_model` path — do not recolor per provider
- **Never** style or print access tokens, refresh tokens, raw JWTs, or full `Authorization` headers

## Color

60/30/10 against **GrokNight** defaults (other themes inherit role mapping, not hard hex):

| Role | Token (GrokNight) | Approx | Usage |
|------|-------------------|--------|-------|
| Dominant (60%) | `bg_base` / terminal bg | `#141414` | List/modal background, empty space |
| Secondary (30%) | `bg_highlight` / `bg_hover` / `bg_visual` | `#242424` / `#2c2c2c` / `#363636` | Selected row, hover, popup panels |
| Accent (10%) | `accent_model` + `fuzzy_accent` | teal `#1abc9c` / blue `#7aa2f7` | Prompt model name + fuzzy match only (unchanged from Phase 3) |
| Destructive / gate fail | `accent_error` | `#f7768e` | Missing-provider **block** system/error emphasis; failed switch scrollback; ACP user-visible failure line if shown in TUI |

**Accent reserved for (explicit — never “all interactive elements”):**

1. Current model name in the prompt status / info line (`accent_model`) — including after a successful switch
2. Fuzzy-match character highlights inside the model picker search (`fuzzy_accent`)
3. **Gate/block failures** use `accent_error` only for error emphasis (scrollback fail line or error-styled toast) — **not** for the whole picker list
4. Success switch confirmation stays **system scrollback primary text** (existing pattern); do **not** paint success with `accent_error` or invent a new success Theme role this phase

**Provider coloring:** **none this phase.** Providers remain plain text labels (`xAI`, `Codex`). Auth badge is dim secondary, not a brand color.

**Text hierarchy (Phase 6 surfaces):**

| Content | Color |
|---------|-------|
| Unselected model name | `text_primary` |
| Selected model name | `text_primary` + bold + selected row bg |
| Auth badge `needs login` | `gray` / `text_secondary` |
| Gate modal question | `text_primary` |
| Gate option descriptions / CLI fallback | secondary / dim |
| Switch success scrollback | system block primary (existing) |
| Switch fail / missing-provider | system or toast with `accent_error` emphasis when error-styled path is used |
| Secrets | **never displayed** |

## Copywriting Contract

### Provider labels (canonical — match Phase 3 / 5)

| Wire id | User-visible label |
|---------|-------------------|
| `xai` | `xAI` |
| `codex` | `Codex` |

**Product command name in copy:** always `bum` for all **new** Phase 6 strings.

### Primary CTA

| Surface | Copy |
|---------|------|
| Primary CTA (gate recovery) | **Login now** |
| Secondary / dismiss | **Keep current model** — description: `Dismiss and stay on the active model` |
| Successful switch (no extra CTA) | N/A — confirmation is informational only |

### Missing-provider gate modal (TUI — required)

Reuse **QuestionView** (same family as agent-type mismatch). Single question, two options, no freeform.

| Element | Copy |
|---------|------|
| Question | `Sign in to {ProviderLabel} to use {ModelDisplayName}.` |
| Option 1 label | `Login now` |
| Option 1 description | `Run login for {ProviderLabel} (CLI: bum login --provider {id})` |
| Option 2 label | `Keep current model` |
| Option 2 description | `Dismiss and stay on the active model` |
| Always-visible CLI fallback | Command form must appear in option description: `bum login --provider {id}` where `{id}` is `xai` or `codex` |
| Secrets | Never show tokens, auth.json body, or bearer material |

**Examples (locked templates):**

- Codex missing, target Sol:  
  `Sign in to Codex to use GPT-5.6 Sol (Codex).`  
  Login now · `Run login for Codex (CLI: bum login --provider codex)`
- xAI missing, target Grok Build:  
  `Sign in to xAI to use Grok Build (xAI).`  
  Login now · `Run login for xAI (CLI: bum login --provider xai)`

**`{ModelDisplayName}`:** catalog `name` string (already includes provider suffix from Phase 3). Do not invent a second provider token in the question beyond the explicit “Sign in to {ProviderLabel}” clause.

### Auth badge on picker rows

| Element | Copy |
|---------|------|
| Badge text | `needs login` (exact; lowercase; no punctuation) |
| Placement | **Right-label** when the surface supports `right_label`; otherwise trailing suffix on the primary display string after a middle-dot separator: ` · needs login` |
| When shown | Target model’s `provider` has **no usable credentials** (`has_usable_token` / equivalent false). Expired-but-refreshable = **usable** → **no badge** |
| When hidden | Provider has usable creds; or same-provider rows when that provider is logged in |
| Current model row | If somehow current while unusable (edge), still show badge; normal path never commits unusable provider as current |
| Selectability | Rows with badge remain **fully selectable** — selecting runs the gate (does not gray-out or skip) |
| Description line | Keep catalog description; do **not** replace description with the badge (badge is separate light cue) |

**Slash `/model` current suffix** (Phase 3) still applies:  
`{name} (current)` — if also needs login (rare): `{name} (current) · needs login` or name+(current) with right-label badge — prefer **right-label** when available so `(current)` stays on the left label.

### Successful switch confirmation

| Element | Copy |
|---------|------|
| Unchanged model+effort | **No message** (existing lifecycle) |
| Same- or cross-provider success | Scrollback system: `Switched to {ModelDisplayName}` |
| With effort | `Switched to {ModelDisplayName} ({effort} effort)` |
| Cross-provider note | **No extra modal.** Provider is already in `{ModelDisplayName}` via Phase 3 suffix. Do not spam a second “now using Codex” toast |
| Mid-turn pending apply | No additional copy required; next turn uses new model. Do **not** cancel in-flight turn or show a blocking “wait” modal |

### Failure / empty / error copy

| Element | Copy |
|---------|------|
| Empty state heading | N/A for gate — gate only appears after a selection. Catalog empty remains Phase 3: slash returns no dropdown |
| Empty state body | N/A (gate is selection-driven) |
| Gate cancel | No toast required; stay on previous model silently |
| Login abandoned / fail | Reuse Phase 5 error strings for the login path; after fail, **do not** apply pending model; optional toast: `Login failed — still on {CurrentModelDisplayName}.` |
| Post-login auto-retry success | Same as successful switch: `Switched to {ModelDisplayName}` |
| Post-login auto-retry fail (other) | `Couldn't switch model: {safe_detail}` (existing Other path) — never include tokens |
| Generic switch Other | `Couldn't switch model: {msg}` (existing) |
| Unknown model (`/model`) | `Unknown model: {input}` (Phase 3; unchanged) |
| Usage (`/model` empty args) | `Usage: /model <name> [effort]` (Phase 3; unchanged) |
| Destructive confirmation | **none this phase** — Keep current model is non-destructive; login is not destructive |

### ACP / headless typed error (machine-readable)

Separate typed path from `IncompatibleAgent` / `MODEL_SWITCH_INCOMPATIBLE_AGENT`.

| Field | Contract |
|-------|----------|
| Stable code | `MODEL_SWITCH_MISSING_PROVIDER` |
| Payload shape | camelCase JSON in `acp::Error.data` (mirror IncompatibleAgent pattern) |
| Required fields | `code`, `provider` (`xai` \| `codex`), `modelId`, `suggestion` |
| `suggestion` | Exact CLI: `bum login --provider {id}` |
| Human `message` (ACP error message) | `Cannot switch to model '{modelId}': no usable {ProviderLabel} credentials. Run: bum login --provider {id}` |
| TUI mapping | Deserialize → open missing-provider QuestionView (not agent-type mismatch; not generic Other string-only) |
| Never | Collapse into `IncompatibleAgent`; never silent 401 as primary UX for switch |

**Suggested struct fields (contract names):**

```text
code: "MODEL_SWITCH_MISSING_PROVIDER"
provider: "xai" | "codex"
modelId: string
suggestion: "bum login --provider xai" | "bum login --provider codex"
```

### CLI-only note (if switch attempted outside TUI)

Headless/CLI that can set model must surface the same provider + suggestion; no raw stack traces as primary UX.

## Interaction Contract

### Gate timing (MOD-06)

| Rule | Contract |
|------|----------|
| When | **At switch time** — before applying the new model on picker confirm, `/model` submit, settings default-model live switch, and ACP `session/set_model` |
| Usable credentials | Existing `has_usable_token` (or equivalent): present and not permanently unusable; **expired-but-refreshable is OK** |
| Same-provider switch | **No extra friction** when target provider has usable creds (e.g. Sol→Terra with Codex logged in) |
| Fail closed | Block switch; **do not** apply model; **do not** leave optimistic current-model on the forbidden id |
| Optimistic UI | **No optimistic `models.current` update** until the gate passes. (Differs from any legacy optimistic+rollback path for this error class — missing-provider never flashes the target as current.) |
| Primary UX | Modal gate + login prompt — **not** silent refuse and **not** mid-turn 401 as design center |
| Secondary | Mid-session token death after a successful switch remains secondary recovery (existing sample/auth paths); not the MOD-06 center |

### Entry points (all honor the same policy)

| Surface | Behavior |
|---------|----------|
| Model picker confirm | Gate → pass apply \| fail modal |
| `/model` / `/m` | Same gate after resolve by name/id |
| Settings → Default model (if it also switches active session) | Same gate before active-session apply |
| ACP `session/set_model` | Same gate in shell `model_switch::apply` (single authoritative check preferred) |
| Mid-turn switch | **Allowed**; apply for **next turn only**; do **not** cancel in-flight turn by default |

### Login recovery

| Rule | Contract |
|------|----------|
| Login now | Reuse Phase 5 provider login (browser PKCE and/or device) for the **missing provider only** |
| CLI always listed | Option description always includes `bum login --provider {id}` |
| Pending switch | On Login now, stash target model (+ effort if any) via **`deferred_model_switch`** (or equivalent) so user does not re-pick |
| After successful login | **Auto-retry** pending switch once credentials usable |
| After cancel | Clear pending gate intent; keep previous model; no auto-login |
| After login failure | Keep previous model; keep or clear deferred per implementer discretion but **never** apply unusable target |
| Isolation | Credentials only under `$BUM_HOME` — never stock `~/.codex` / `~/.grok` |

### Picker presentation (auth-aware list)

| Rule | Contract |
|------|----------|
| Catalog shape | **Full mixed list** always (Phase 3) — never hide GPT rows when Codex logged out |
| Badge | Light `needs login` only — see Copywriting |
| No global provider filter | Still no session-global “Codex mode” |
| History | **Keep full conversation history** across cross-provider switches in the same session |

### Separation from IncompatibleAgent

| Path | UI |
|------|----|
| Missing provider | QuestionView: Login now / Keep current model; typed `MODEL_SWITCH_MISSING_PROVIDER` |
| Incompatible agent | Existing QuestionView: Yes (new session) / No; typed `MODEL_SWITCH_INCOMPATIBLE_AGENT` |
| Other | Scrollback `Couldn't switch model: {msg}` |

Do **not** collapse these into one generic modal or one error code.

### Explicit non-goals (defer)

- Cross-provider subagent spawn gate UI → **Phase 7**
- Full rebrand of remaining Grok chrome strings → **Phase 8**
- Per-provider brand colors / icon badges → later polish
- Filtering catalog by auth status → never (conflicts with Phase 3)
- Stock credential import → never v1
- Cancelling in-flight turn on model switch → not default this phase

## UI Considerations

> Shape-rooted state coverage for switch/gate surfaces. Empty/error **copy** lives in
> **Copywriting Contract** — this table references those rows only.

Applicable state considerations resolved: **8 covered, 0 backstop, 0 unresolved**

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | model list-collection with no auth badges needed | ✅ covered | Full catalog still listed (Phase 3). No badge when all relevant providers usable. Gate modal does not appear without a selection. |
| loading | login recovery after Login now | ✅ covered | Reuse Phase 5 waiting copy for browser/device flows (`Waiting for authorization...` / browser wait). No new spinner chrome required. Gate modal may close or stay blocked until login settles; pending model not applied until usable. |
| error | missing-provider gate; login fail; generic switch Other; ACP typed error | ✅ covered | QuestionView for missing provider; login fail keeps current model + optional toast; Other → `Couldn't switch model: {msg}`; ACP `MODEL_SWITCH_MISSING_PROVIDER` with suggestion. Separate from IncompatibleAgent modal. |
| populated | picker with mixed usable/unusable providers; successful switch | ✅ covered | Badged rows for unusable providers only; full list; success scrollback `Switched to {ModelDisplayName}`. |
| partial | one provider logged in, other not | ✅ covered | Badge only on rows of the missing provider; switches within logged-in provider have no gate friction; switch to other opens gate. |
| overflow | long model display name + `needs login` badge; long ACP message | ✅ covered | Prefer truncating family name before dropping provider suffix (Phase 3); badge/right-label truncates last if width exhausted — keep badge readable (`needs login` is short). Terminal wrap OK for ACP/CLI messages. |
| zero-one-many | 0 / 1 / many models needing login | ✅ covered | Zero unusable: no badges, switches free when creds OK. One provider unusable: all that provider’s rows badged. Many: each unusable-provider row badged independently. |
| long-text | question string with long model name; CLI suggestion | ✅ covered | QuestionView wraps existing; CLI command never mid-truncated mid-token; model display uses catalog name as-is with natural wrap. |
