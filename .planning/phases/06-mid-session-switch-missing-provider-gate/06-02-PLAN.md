## Phase Goal

**As a** bum user mid-session, **I want to** switch models freely and get a clear fail-closed login path when a provider is missing, **so that** I never depend on silent mid-turn 401 as the primary UX and never restart the CLI to change models.

<objective>
Map shell MODEL_SWITCH_MISSING_PROVIDER into pager TEA: typed SwitchModelError, QuestionView Login now / Keep current model (copy per D-08 / UI-SPEC), and enforce no optimistic current-model on this error class (D-03 modal surface, D-05, D-07).

Purpose: User-visible fail-closed gate after Plan 01 shell authority (MOD-06 TUI half without full login auto-retry yet).
Output: QuestionView opens on blocked switch; Keep current works; settings rollback on MissingProvider.
</objective>
