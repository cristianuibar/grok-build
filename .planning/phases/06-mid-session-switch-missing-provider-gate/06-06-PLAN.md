## Discovery assert pattern (mandatory)

Every PHASE-GATE filter group must:

```bash
set -euo pipefail
discover() {
  local pkg="$1" filter="$2"; shift 2
  local extra=("$@")
  local n
  n=$(cargo test -p "$pkg" "${extra[@]}" -- --list 2>/dev/null | grep -c "$filter" || true)
  test "$n" -ge 1
  cargo test -p "$pkg" "${extra[@]}" "$filter" -- --nocapture
}
```

Do **not** use bare filters that already match unrelated tests as the only Phase 6 proof (`auth_complete`, bare `needs_login` without `p6_`, `deferred_model_switch`). Aggregate `grep -c p6_` alone is **insufficient** for Task 2 verify (cycle 2).

**Allowed file edits in this plan:** documentation only under `.planning/phases/06-mid-session-switch-missing-provider-gate/`. If a test rename is required solely for discoverability of a greened SUMMARY filter, permit a **one-line rename in the relevant test file** listed in that plan's SUMMARY — record the rename in PHASE-GATE notes. Prefer fixing VALIDATION to match landed p6_ names first.
