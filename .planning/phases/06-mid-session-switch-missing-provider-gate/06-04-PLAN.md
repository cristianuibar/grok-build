## Phase Goal

**As a** bum user mid-session, **I want to** switch models freely and get a clear fail-closed login path when a provider is missing, **so that** I never depend on silent mid-turn 401 as the primary UX and never restart the CLI to change models.

<objective>
Add light picker auth cues: authoritative dual-slot usable producer (AuthMeta/status), AppView cache, and `needs login` badge without filtering the mixed catalog (per D-04, D-08, RESEARCH A4). Addresses review HIGH: AuthMeta must expose providers.xai/codex usable flags.

Purpose: Users see which models will gate before selecting; catalog stays complete (Phase 3); Plan 03 reuses the same status producer for external-login deferred apply.
Output: Badge on unusable-provider non-BYOK rows; cache refresh on auth lifecycle; unit tests for display strings.
</objective>

## Authoritative dual-slot producer (contract)

1. **Shell AuthMeta extension** (serde camelCase, defaults safe for old clients):
   - Add nested structure or explicit fields for dual-slot usability, e.g. `providers: Option<ProviderAuthMetaSlots>` with `xai.usable: bool`, `codex.usable: bool` (booleans only — never tokens, refresh tokens, or raw keys).
   - Populate when building AuthMeta (mvp_agent authenticate / session auth meta paths) from `AuthStatusReport::from_auth_file` / `store_usable` semantics (D-02). Missing file → both false.
   - Keep existing xAI account/subscription fields unchanged.
2. **Pager consumer:** `AppView::apply_auth_meta` updates dual-slot cache from these fields when present.
3. **Explicit refresh Effect** (also used by Plan 03): `Effect::RefreshProviderAuthStatus` loads AuthStatusReport (or shell-exported helper) and applies pure booleans to AppView. Startup: one-shot via existing auth meta path OR refresh effect.
4. Stale/error behavior: on refresh IO/parse failure, keep last known cache (or fail-closed both false) — document choice; never log auth file contents.

Do **not** edit dispatch/auth.rs Login-now body here (Plan 03 owns deferred apply). Expose public cache + refresh helpers Plan 03 can call.
