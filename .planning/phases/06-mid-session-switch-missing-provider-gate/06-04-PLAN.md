## Phase Goal

**As a** bum user mid-session, **I want to** switch models freely and get a clear fail-closed login path when a provider is missing, **so that** I never depend on silent mid-turn 401 as the primary UX and never restart the CLI to change models.

<objective>
Add light picker auth cues: authoritative dual-slot usable producer (AuthMeta/status), AppView cache with **owned lifecycle refresh triggers**, and `needs login` badge on slash **and** settings DynamicEnum without filtering the mixed catalog (per D-04, D-08, RESEARCH A4, UI-SPEC component inventory). Addresses review cycle 2 HIGH residual H3: refresh trigger ownership.

Purpose: Users see which models will gate before selecting; catalog stays complete (Phase 3); Plan 03 reuses the same status producer for external-login deferred apply.
Output: Badge on unusable-provider non-BYOK rows; cache non-default after dual-slot fixtures via wired startup/login/logout/focus refresh; unit tests for display strings + refresh emission.
</objective>

4. **Stale/error behavior:** on refresh IO/parse failure, keep last known cache (or fail-closed both false) — document choice in SUMMARY; never log auth file contents.

Do **not** edit Login-now deferred apply body here (Plan 03). Expose public cache + refresh helpers Plan 03 can call.
