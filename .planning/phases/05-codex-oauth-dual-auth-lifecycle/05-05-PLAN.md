| ID | Decision |

| D-08 | Thin Codex handle + shared TokenRefresher; do not rewrite xAI AuthManager for dual ownership |
| D-11 | organization_id carries chatgpt_account_id |

**Discretion locks:**
- Thin `CodexRefresher` implementing `TokenRefresher`; fixed POST `{issuer}/oauth/token` grant_type=refresh_token (no OIDC discovery)
- Disk-sibling adopt on invalid_grant before wipe (mirror OidcRefresher::retry_with_fresh_disk_token)
- Proactive policy: JWT exp ≤ now+5m first; optional last_refresh field only if required (prefer exp)
- `invalidate_codex_session_key_cache()` public/crate API called from login/logout/refresh persist paths
- ChatGPT-Account-ID inject on Codex sampling config / inject_url_derived_headers or prepare path when organization_id present — only for Codex provider/base, never on xAI
