## Phase Goal

**As a** bum user, **I want** Codex and xAI tokens to refresh independently on every live request, **so that** a ChatGPT session mid-session expiry never logs me out of xAI and never serves a stale prepared bearer.

<objective>
**CRITICAL REDESIGN (Codex review cycle 1 + residual cycle 2):** Implement independent Codex access-token refresh with lock-held RT rotation, pure CodexRefresher, and **async per-request invoker on `SessionActor::reconstruct_full_config`** (sampler_turn.rs), returning typed `{ bearer, account_id }` into request construction (AUTH-05; D-07, D-08, D-11). Cycle 2: gate OAuth bearer override (SessionToken + session_oauth_allowed), permanent-fail clear with held lock only, prove wiring via **Option C reconstruct seam**, preserve identity when IdP omits refresh_token. Do **not** treat ModelsManager prepare/snapshot or ensure_fresh-in-isolation as the sole AUTH-05 proof.

Purpose: Live dual-session health without cross-wipe, BYOK/custom-host safety, no refresh deadlock.
Output: GREEN isolation + reconstruct mid-session (seam) + BYOK/custom endpoint + concurrent single spend + identity preserve + transient policies + trusted account header.
</objective>

   Preserve model-owned BYOK `creds.api_key` unchanged. Do **not** call IdP refresh for BYOK or custom non-allowlisted Codex routes. ChatGPT-Account-ID already trusted-host gated; **bearer must use the same trust rules**.
3. **Executable AUTH-05 wiring seam (cycle 2 HIGH — Option C locked):** Prove reconstruct invokes ensure_fresh by crate-local **SessionActor** tests under `session/acp_session_tests/codex_reconstruct_refresh_tests.rs`, registered like `auth_error_no_retry_tests.rs`, calling `actor.reconstruct_full_config().await` and asserting `SamplingConfig.api_key` / headers. Full shell `--lib` suite remains forbidden as a phase gate; **narrow** `--lib` TESTNAME filters for named reconstruct-wiring tests are the Option C exception.
4. **Lock span:** Under in-process refresh mutex, acquire `auth.json.lock`; re-read `providers.codex`; adopt sibling rotated token if fresher; keep live guard across IdP refresh; persist with guard-held `mutate_provider_store_or_prune_with_lock(AuthProvider::Codex, …)` — mirror AuthManager refresh at manager.rs ~1618-1651 / 1693-1725.
5. **Permanent failure clear (cycle 2 HIGH):** While ensure_fresh holds `auth.json.lock`, permanent fail must **not** call public acquiring `clear_provider_slot` (deadlock). Use `clear_provider_slot_with_lock` (Plan 02) **or** `mutate_provider_store_or_prune_with_lock` with the existing guard. Never reacquire.
6. **Return type:** `CodexAuthMaterial { bearer: String, account_id: Option<String> }` returned into reconstruct for `api_key` + header inject — do not rely only on mtime cache invalidation of prepare-time stamps.
7. **CodexRefresher purity:** Implements `TokenRefresher`; returns `RefreshOutcome` only; **never** mutates store (Addresses review: trait contract at refresh/mod.rs ~86-143). Storage ops live only in ensure_fresh / refresh_chain-equivalent outer function.
8. **Identity preservation on refresh (cycle 2 MEDIUM):** When IdP refresh response omits `refresh_token` and/or account claims, preserve old refresh token, `organization_id`/ChatGPT account id, email, issuer, and client_id (mirror `oidc/refresh.rs` ~189–212). Add mock omitting those fields.
9. **Near-expiry policy:** JWT exp or expires_at ≤ now+5m triggers refresh when refresh_token present; fresh tokens make zero IdP calls.
10. **Transient failure:** hard-unexpired access → return old material; hard-expired → return None (no credential); never clear either provider on transient errors.
11. **Permanent failure scope:** clear/mark Codex only; never AuthManager::clear; never touch xAI.
12. **Account header:** inject `ChatGPT-Account-ID` only when routed provider is Codex, credential is ChatGPT OAuth material, and endpoint is trusted ChatGPT Codex backend — **not** via global auth read inside bare inject_url_derived_headers for arbitrary hosts.
13. Injectable HTTP / TokenRefresher for tests; public or test-facing ensure_fresh for isolation/concurrency; reconstruct seam for production wiring.

## Production path sketch (normative)

```
reconstruct_full_config (async):
  cfg, creds, model_facts = ...
  if model_facts.provider == Some(Codex)
     AND creds.auth_type == SessionToken
     AND session_oauth_allowed(base_url / route):
    material = ensure_fresh_codex_auth(...).await  // lock + maybe IdP
    api_key = material.as_ref().map(|m| m.bearer)  // override prepared session stamp only
    maybe inject ChatGPT-Account-ID from material.account_id if trusted host
    bearer_resolver = None  // Codex not xAI AuthManager
  else if Codex BYOK or custom host (not session OAuth eligible):
    keep prepared creds.api_key; do not ensure_fresh / do not attach session OAuth
  else if Xai attach policy:
    existing AuthManager bearer resolver path
  inject_url_derived_headers for proxy only — account id not from URL alone

ensure_fresh permanent fail (holding AuthFileLock):
  clear_provider_slot_with_lock(auth_file, &lock, Codex)  // NEVER clear_provider_slot
```

Do not expand this sketch into fenced implementation code inside task actions.
