## Locked decisions (traceability)

| ID | Decision |
|----|----------|
| D-07 | Independent refresh chains; permanent fail slot-only |
| D-08 | Thin Codex handle + shared TokenRefresher; do not rewrite xAI AuthManager for dual ownership |
| D-11 | organization_id carries chatgpt_account_id |
| D-13 | Mock HTTP refresh proofs |

**Discretion locks:**
- Thin `CodexRefresher` implementing `TokenRefresher`; fixed POST `{issuer}/oauth/token` grant_type=refresh_token (no OIDC discovery)
- Disk-sibling adopt on invalid_grant before wipe (mirror OidcRefresher::retry_with_fresh_disk_token)
- **Proactive / prepare policy:** JWT exp or `expires_at` ≤ now+5m triggers refresh on prepare path; prefer exp (no new last_refresh field)
- **Production invoker (mandatory):** prepare-time / pre-request path — not cache invalidate alone. Preferred entry: `ensure_fresh_codex_session_key` (name discretionary) called from ModelsManager prepare / `snapshot_codex_session_key_from_auth_store` callers / `prepare_sampling_credentials` adjacent path so every Codex sample can refresh near-expiry tokens
- `invalidate_codex_session_key_cache()` public/crate API called from login/logout/refresh persist paths
- ChatGPT-Account-ID inject on Codex sampling config / inject_url_derived_headers or prepare path when organization_id present — only for Codex provider/base, never on xAI
- Injectable HTTP / TokenRefresher in tests so prepare-path lifecycle can mock IdP without network
