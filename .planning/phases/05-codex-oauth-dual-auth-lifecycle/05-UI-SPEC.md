# Phase 5 — UI Design Contract

> Visual and interaction contract for **Codex OAuth & dual auth lifecycle** surfaces.
> Product is a **ratatui terminal TUI + CLI**, not a web app. Tokens map to
> `Theme` fields and character-cell layout (TUI) or plain stderr/stdout (CLI), not CSS/shadcn.

## Typography

Terminal monospaced — “size” is **role weight + Theme/ANSI**, not px. Exactly **3 roles**, **2 weights**.

| Role | Size (cells) | Weight | Line Height | Theme / style |
|------|--------------|--------|-------------|---------------|
| Body (primary CLI/TUI message) | 1 row | regular (400) | 1.0 | default stderr/stdout; TUI `text_primary` |
| Label (section headers, provider names, field keys) | 1 row | **bold when TUI-selected only**; CLI regular | 1.0 | Provider human labels plain; status keys before `:` |
| Secondary (safety notes, hints, waiting) | 1 row | regular (400) | 1.0 | CLI dim ANSI (`\x1b[90m`…`\x1b[0m`); TUI `gray` / `text_secondary` |
| Display | — | — | — | **not used this phase** (no new modal titles) |

**Rules:**

- Max **2** font weights: regular + bold (TUI selected row only if any list appears)
- User **code** printed as body on its own indented line — never dimmed (must be copyable and readable)
- Success prefix `✓` stays plain text (no emoji variants)
- **Never** style or print access tokens, refresh tokens, raw JWTs, `client_secret`, or full `Authorization` headers

## Copywriting Contract

### Provider labels (canonical — match Phase 3)

| Wire id (`--provider`) | User-visible label |
|------------------------|--------------------|
| `xai` | `xAI` |
| `codex` | `Codex` |

**Product command name in copy:** always `bum` (not `grok`) for Phase 5 user-facing auth strings that are **new or rewritten**. Pre-existing xAI-only strings that still say `grok` may remain until Phase 8 unless the line is touched for dual-provider work — then rewrite to `bum`.

### CLI: login (`bum login`)

| Element | Copy |
|---------|------|
| Primary CTA | **Sign in** (verb) — flow is the CTA; no separate button |
| Bare login | `bum login` → **xAI only** (Phase 2 lock). Do not prompt for provider. |
| Provider flag | `bum login --provider codex` \| `bum login --provider xai` |
| Device flags | `--device-auth` (alias `--device-code`) forces device-code flow |
| Start (browser PKCE, CLI) | `Signing in with {ProviderLabel}...` then blank line + `Open this URL to sign in:` + indented URL |
| Start (device-code, CLI) | See **Device-code display** block below |
| Waiting (browser) | Implicit while loopback listens; optional secondary: `Waiting for browser authorization...` if a status line is needed |
| Waiting (device) | `Waiting for authorization...` |
| Success (with email) | `✓ Signed in to {ProviderLabel} as {email}` |
| Success (no email) | `✓ Signed in to {ProviderLabel}` |
| Re-login | Overwrites **only** that provider’s slot; success copy same as first login |
| Empty / no session pre-login | N/A (login always starts a flow) |
| Error (unknown provider) | `Unknown provider: {input}. Use --provider xai or --provider codex.` |
| Error (device expired) | `Device code expired. Run \`bum login --provider {id} --device-auth\` again.` |
| Error (auth denied) | `Authorization denied. The user rejected the request.` |
| Error (generic exchange) | `Token exchange error: {safe_detail}` — **never** include tokens in `{safe_detail}` |
| Error (port bind fail) | `Could not bind OAuth callback on localhost:1455 (or 1457). Free the port or use --device-auth.` |
| Destructive confirmation | **none on login** |

**Codex-specific start note (browser):** provider label in “Signing in with Codex...” — do not say “OpenAI Platform” or imply API-key billing.

### CLI: logout (`bum logout`)

| Element | Copy |
|---------|------|
| Primary CTA | **Log out** of a named provider (or all when explicit) |
| Selective | `bum logout --provider codex` \| `--provider xai` |
| Dual clear | `bum logout --all` only |
| Bare logout | **Fail closed** — do **not** silent dual-wipe. Print usage and exit non-zero (or 2 for usage): |
| Bare logout body | `Specify a provider or clear all:\n  bum logout --provider xai\n  bum logout --provider codex\n  bum logout --all` |
| Success selective (with email) | `Logged out of {ProviderLabel} (was signed in as {email})` |
| Success selective (no email) | `Logged out of {ProviderLabel}` |
| Success `--all` | `Logged out of all providers` (optionally list which were cleared: `Logged out of xAI and Codex`) |
| Empty (not logged in, selective) | `No cached {ProviderLabel} session to log out of.` |
| Empty (`--all`, none logged in) | `No cached sessions to log out of.` |
| API key still set (xAI path only) | Keep existing pattern when xAI env key present: `XAI_API_KEY is still set and will be used for authentication.` — only after xAI logout / when relevant; never invent a Codex API-key primary warning |
| Destructive confirmation | **CLI flags are the confirmation** — no interactive `y/N` prompt this phase. `--all` is the explicit dual-destructive affordance. |

### CLI: status (`bum auth status`)

| Element | Copy |
|---------|------|
| Primary CTA | **Inspect auth** (read-only) |
| Command | `bum auth status` (first-class; equivalent nesting OK if clap prefers, but user-facing docs use this form) |
| Header | `Auth status` (optional single line) |
| Per-provider block | See format below |
| Empty both | Both providers still listed with `logged_in: no` — never hide a provider row |
| Error (store unreadable) | `Could not read auth store at {path_display}. Fix permissions or re-run login.` — path may show `$BUM_HOME/auth.json` style label; **never** dump file contents |
| Secrets | **Never** print `access_token`, `refresh_token`, `key`, raw JWT, or full auth.json |

**Status block format (locked, greppable, paste-safe):**

```
xAI:
  logged_in: yes|no
  usable: yes|no
  account: {email_or_—}
  plan: {plan_or_—}

Codex:
  logged_in: yes|no
  usable: yes|no
  account: {email_or_—}
  plan: {plan_or_—}
```

| Field | Meaning |
|-------|---------|
| `logged_in` | Slot has a session record (token or equivalent present) |
| `usable` | Credentials present and not known-invalid / permanently failed refresh |
| `account` | Email or stable account label from claims when available; otherwise `—` |
| `plan` | Plan/tier label when claims exist; otherwise `—` |

**Order:** xAI block first, then Codex. One blank line between blocks. No third provider rows in v1.

### Device-code display (CLI stderr)

Mirror existing xAI device UX shape; Codex verification URL from STACK contract:

| Element | Copy |
|---------|------|
| Open URL | `To sign in, open this URL in your browser:` |
| URL line | `  {verification_uri or verification_uri_complete}` |
| Browser fail | `  (Could not open browser automatically — open the URL above manually.)` |
| Code prompt (complete URI present) | `Confirm this code in your browser:` |
| Code prompt (type code) | `Then enter this code:` |
| User code line | `  {user_code}` — full code visible, no masking |
| Safety | dim: `Only continue with a code you requested. Don't share it with anyone.` |
| Waiting | `Waiting for authorization...` |
| Codex verification URL default | `https://auth.openai.com/codex/device` (plus complete URI when server provides it) |

### Browser PKCE waiting (CLI stderr)

| Element | Copy |
|---------|------|
| Provider start | `Signing in with {ProviderLabel}...` |
| Open URL | `Open this URL to sign in:` |
| URL indent | `  {authorize_url}` — may be long; do not truncate mid-query if terminal wraps (wrapping OK) |
| Manual paste hint (TTY) | `Paste the URL here if it doesn't connect:` |
| Waiting | No spinner required; process blocks until callback/timeout. Optional: `Waiting for browser authorization...` |
| Cancel / abandon | No special success line; process exits with error if flow fails. Do not clear the *other* provider on abandon. |

### TUI touchpoints (minimal — non-blocking)

| Element | Copy / behavior |
|---------|-----------------|
| Primary CTA | Existing welcome **Sign in** path may remain xAI-default this phase |
| `/logout` description | Prefer: `Log out of a provider (CLI: bum logout --provider …)` if description updated; minimum: must **not** imply dual-wipe |
| `/logout` usage | `/logout` — if TUI still clears only legacy/default xAI slot, document that; **must not** clear both slots without provider choice |
| TUI dual-login chrome | **Out of scope** unless trivial; do not block phase on provider picker in welcome |
| Empty / missing Codex in TUI | No Phase 6 gate copy here — defer “login to Codex to use GPT” to Phase 6 |
| Error in TUI auth | Reuse existing auth error surface; map permanent Codex failure to re-login hint: `Codex session expired. Run bum login --provider codex.` |
| Destructive `/logout` | Prefer fail-closed message if bare `/logout` would dual-wipe: show toast/scrollback `Use bum logout --provider xai|codex or --all` rather than wiping both |

### Secrets & support paste policy (global)

| Rule | Contract |
|------|----------|
| Tokens | Never print access/refresh tokens or `key` fields |
| auth.json | Never dump raw file to stdout/stderr on status or error |
| Logs | Refresh/error tracing may log lengths or error codes only (`api_key_len`, `invalid_grant`) — not bearer material |
| Status paste | Output must be safe to paste into support tickets |

## Interaction Contract

### Login entry (AUTH-02)

| Rule | Contract |
|------|----------|
| Bare `bum login` | xAI OAuth only (browser default; `--device-auth` for device) |
| `--provider codex` | Codex ChatGPT OAuth PKCE (browser default) or device with `--device-auth` |
| `--provider xai` | Explicit xAI (same as bare) |
| Re-login | Overwrites **only** targeted provider slot |
| Isolation | Credentials only under `$BUM_HOME/auth.json` — never `~/.codex` / `~/.grok` |
| Product path | ChatGPT OAuth primary — not Platform API key marketing |

### Logout (AUTH-03)

| Rule | Contract |
|------|----------|
| Selective | Clears one slot only |
| `--all` | Clears both slots when explicit |
| Bare | Fail closed with usage (no silent dual-wipe) |
| Independence | Logging out Codex leaves xAI intact and vice versa |

### Status (AUTH-04)

| Rule | Contract |
|------|----------|
| Always both rows | xAI + Codex always listed |
| Greppable | `logged_in:` / `usable:` keys stable for scripts |
| No secrets | Paste-safe |

### Independent refresh (AUTH-05) — UI-visible effects only

| Rule | Contract |
|------|----------|
| Silent success | No user message on routine proactive refresh |
| Permanent failure (one provider) | User-facing error or next-sample failure references **that** provider only; other provider remains usable |
| Copy on permanent fail | `{ProviderLabel} session is no longer valid. Run \`bum login --provider {id}\`.` |
| Never | “Logged out of all providers” as side effect of one-sided refresh failure |

### Explicit non-goals (defer)

- Mid-session model-switch login gate → **Phase 6**
- Rich dual-provider welcome / account switcher chrome → Phase 6/8
- Full string rebrand of untouched Grok labels → **Phase 8**
- Import stock `~/.codex` credentials → never v1
- Interactive `y/N` confirm on `--all` → not required this phase
