## Required design (C2-H1 — non-negotiable)

**REQUIRED production mechanism:** async `SubagentBackend::preflight_spawn` (oneshot RPC via `ChannelBackend` / `SubagentEvent`, same pattern as `validate_type` / `describe_subagent_type`) handled in the shell coordinator with **live parent session state**.

The handler MUST:
1. Resolve effective child model with the **same async path** as `handle_subagent_request` / `resolve_effective_model_config` — including omit-model inherit via live `ChatStateHandle` parent sampling config and role/config model pins (not a model id snapshot frozen at `AgentRebuildSpec::build_agent`).
2. Map effective model → catalog provider → Plan 03 pure `provider_slot_usable` / `missing_provider_gate_error`.
3. Return Ok/None (allow) or Err/Some(login-shaped message) without creating worktree, pending child, or starting spawn.

**NOT sufficient / NOT the production default (document explicitly in SUMMARY):**
- Sync `Arc<dyn Fn(slug) -> Option<String>>` or sync `Fn(TaskCredentialPreflightRequest) -> Option<String>` installed at agent rebuild — cannot query live parent chat state or match async role-pin resolve.
- Slug-only gates that skip when `Task.model` is omitted.
- Treating shell-only gate after `tokio::spawn` as meeting AGENT-05 UX (background already returned "started").

**Allowed equivalent (only if proven):** an async resource/trait object that receives a **live** parent-model source at call time (not a rebuild-time snapshot) and shares Plan 03 pure helpers. If chosen, SUMMARY must prove live-state access for omit-model + role pins. Default remains backend `preflight_spawn`.

**Discretion:** Exact outcome type naming (`SubagentPreflightOutcome`, `Result<(), ToolError>`, mirror of `SubagentValidateTypeOutcome`) and event field layout — follow existing oneshot RPC style in `backend.rs`.

## Scope boundaries

- Do **not** reimplement shell gate logic differently — preflight handler must call the same pure helpers Plan 03 extracted.
- Do **not** open auth.json inside xai-grok-tools (I/O stays in shell handler).
- Do **not** add TUI QuestionView for Task spawn (tool error is primary).
- Do **not** ship a sync slug-only gate as the AGENT-05 solution even if unit-tested.
- No live OAuth.
