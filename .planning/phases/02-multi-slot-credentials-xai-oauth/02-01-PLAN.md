## Phase Goal

**As a** bum user, **I want to** store xAI credentials in a multi-provider auth file under bum home, **so that** a later Codex slot cannot be wiped by xAI login and agent turns still authenticate on the xAI path.

<objective>
Deliver the multi-slot on-disk `auth.json` core: `AuthDocument` schema, legacy flat → `providers.xai` read migration, merge-safe write that emits nested form with optional `version: 1`, and storage unit tests proving migrate + nested write + codex clobber resistance (per D-01, D-02, D-03, D-04, D-05, D-10, D-11, D-14).

Purpose: Dual-OAuth-safe storage is the foundation for AUTH-01 under bum; AuthManager stays storage-agnostic via the xAI-slot adapter (D-12).
Output: Working multi-slot storage layer with automated isolation tests; no Codex login (D-15), no stock-home import (D-13).
</objective>
