## Phase Goal

**As a** bum user, **I want to** clear an xAI API key or log out of xAI without wiping a reserved sibling provider slot, **so that** multi-provider credentials stay isolated under one auth.json.

<objective>
Make API-key helpers and scope-removal empty-delete multi-provider-aware (per D-05, D-09, D-11). After Plan 01 merge-safe writes, empty-xAI paths must not `remove_file` when `providers.codex` (or any sibling) still holds scopes (AUTH-01 isolation / Phase 5 readiness).

Purpose: Prevent the classic empty-map delete pitfall that would log out Codex once Phase 5 fills the slot.
Output: Shared prune-or-delete helper; `store_api_key` / `clear_api_key` / `write_scope_removal` isolation tests green.
</objective>
