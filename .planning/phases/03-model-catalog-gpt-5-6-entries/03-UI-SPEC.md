## Spacing Scale

Declared values for **terminal layout** (cell-based; multiples of 4 where width budgets apply):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 col | Gap before provider badge space / between glyph and label |
| sm | 2 cols | Indent steps, gap before right-label, trailing pad before border |
| md | 4 cols | Section gutter inside modal/popup frames |
| lg | 8 cols | Modal horizontal padding (existing popup frame) |
| xl | 16 cols | Reserved — do not introduce new wide gutters this phase |
| 2xl | 24 cols | Reserved |
| 3xl | 32 cols | Reserved |

**Row geometry (locked to existing picker):**

| Element | Contract |
|---------|----------|
| List row height | 1 cell primary line; optional +N wrapped description lines |
| Fold/prefix width | 2 cells (`◆ ` / `› `) |
| Trailing pad | 1 cell before scrollbar/border |
| Right-label gap | 2 cells when `right_label` non-empty |
| Indent step | 2 cells per indent level |

**Exceptions:**

- Icon-only touch targets (44px) — **N/A** (keyboard/mouse terminal, not mobile web)
- Do not invent CSS rem/px spacing outside the cell table above
- **Character-cell chrome (xs=1 / sm=2, fold/prefix 2 cells, trailing pad 1):** locked to existing picker geometry (`views/picker.rs`). These are **not** web 4px-grid violations — do not "fix" them to 4-col prefixes or you will break fold/glyph layout. Larger gutters (`md`–`3xl`: 4/8/16/24/32) already match the standard set.

## Typography

Terminal monospaced — “size” is **role weight + Theme color**, not px. Exactly **3 roles**, **2 weights**.

| Role | Size (cells) | Weight | Line Height | Theme / style |
|------|--------------|--------|-------------|---------------|
| Body (primary row label) | 1 row | regular (400) / **bold when selected** | 1.0 (single cell row) | `text_primary`; selected + `Modifier::BOLD` |
| Label (provider suffix, meta, CLI bullets) | 1 row | regular (400) | 1.0 | Provider suffix part of label string; CLI `* `/`- ` prefix dim |
| Secondary (description under row) | 1+ wrapped rows | regular (400) | 1.0 | `gray` / `text_secondary` via picker meta/description styles |
| Display | — | — | — | **not used this phase** (no titles beyond existing modal chrome) |

**Rules:**

- Max **2** font weights: regular + bold (selected row only)
- No italic for model names; italic reserved for existing ghost-text elsewhere
- Fuzzy match highlight uses `theme.fuzzy_accent` on matched spans only
- Prompt chrome model name continues to use `theme.accent_model` (teal on GrokNight) — do not recolor per provider this phase

## Color

60/30/10 against **GrokNight** defaults (other themes inherit role mapping, not hard hex):

| Role | Token (GrokNight) | Approx | Usage |
|------|-------------------|--------|-------|
| Dominant (60%) | `bg_base` / terminal bg | `#141414` | List/modal background, empty space |
| Secondary (30%) | `bg_highlight` / `bg_hover` / `bg_visual` | `#242424` / `#2c2c2c` | Selected row, hover row, popup panels |
| Accent (10%) | `accent_model` + `fuzzy_accent` | teal `#1abc9c` / blue `#7aa2f7` | **Only** prompt model name + fuzzy match highlights |
| Destructive | `accent_error` | `#f7768e` | Not used for catalog rows this phase; reserved for future gate/errors |

**Accent reserved for (explicit — never “all interactive elements”):**

1. Current model name in the prompt status / info line (`accent_model`)
2. Fuzzy-match character highlights inside the model picker search (`fuzzy_accent`)

**Provider label coloring:** **none this phase.** Provider is plain text in the name string (` (xAI)` / ` (Codex)`). Do not introduce per-provider accent colors until a later polish phase.

**Text hierarchy:**

| Content | Color |
|---------|-------|
| Unselected model name | `text_primary` |
| Selected model name | `text_primary` + bold + selected row bg |
| Description | `gray` / secondary |
| Dimmed / non-selectable (if any) | `gray_dim` |
| CLI current marker `*` | default stdout; no ANSI theme required for v1 list |

## Interaction Contract

### List structure (MOD-01, MOD-02)

| Rule | Contract |
|------|----------|
| Shape | **Single flat mixed list** — no tabs, no provider filter mode, no session-global provider scope |
| Order | **xAI/Grok entries first** (catalog insertion order), then GPT-5.6 **Sol → Terra → Luna** |
| Binding | Every row’s underlying entry has explicit `provider`: `xai` \| `codex` (for Phase 4; visible via name suffix now) |
| Visibility | GPT rows **always listed** even if Codex credentials missing |
| Default model | Remain **`grok-build`** (`provider: xai`) |
| Scope this phase | Catalog + labels + visibility only — **no** Codex routing, **no** missing-provider block modal |

### Surfaces

| Surface | Behavior |
|---------|----------|
| `/model` / `/m` | Arg dropdown lists all user-selectable models in catalog order; display = name (with provider suffix); description = catalog description when present; resolve by name **or** id |
| Settings → Default model | DynamicEnum from active catalog; inherits **provider-labeled names only** (per-row descriptions may be empty / N/A — no extra description plumbing required this phase); no provider filter |
| `bum models` | Prints default id + flat list with id and provider-bearing name; mark current with `*` |
| ACP `ModelInfo` | `name` already includes provider suffix; **meta key `provider` always set** (`"xai"` \| `"codex"`) for machine consumers alongside existing `agentType`. UI must not depend on meta alone for the visible label — name carries the provider suffix |

### Selection chrome

| State | Visual |
|-------|--------|
| Unselected row | primary text, base bg |
| Selected / keyboard focus | bold primary + `bg_visual` (or embedded modal selected style) |
| Mouse hover | `bg_hover` |
| Current model in slash list | name + ` (current)` in display string |
| Search / filter | existing fuzzy filter; **no matches → generic picker empty list** (no crash). Distinct from **empty catalog** → slash returns no dropdown (`None`) — do not conflate the two states |

### Explicit non-goals (defer)

- Login prompt when picking Codex without auth → Phase 6  
- Per-provider color badges / icons → later polish  
- Capability matrix / effort matrix UI → MOD-V2-01  
- BYOK custom GPT rows as product requirement → deferred

## UI Considerations

> Shape-rooted state coverage for model-list surfaces. Copy for empty/error lives in
> **Copywriting Contract** — this table references those rows only.

Applicable state considerations resolved: **8 covered, 0 backstop, 0 unresolved**

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | model list-collection (`/model`, settings enum, CLI) | ✅ covered | **Authoritative:** slash `suggest_args` returns `None` when `models.is_empty()` (no dropdown). Copywriting “No models available” is N/A for that path. Settings empty enum / CLI empty list: no crash; no provider filter. |
| loading | model list-collection | ✅ covered | Phase 3 catalog is embedded defaults (+ existing remote merge). No dedicated spinner: show last-known/merged catalog immediately; remote prefetch must not strip GPT rows while in flight |
| error | model pick / unknown id | ✅ covered | **Unknown model: {input}** or usage string; list itself does not enter a hard error empty-state for partial remote failure — keep defaults |
| populated | model list-collection | ✅ covered | Flat ordered list: Grok/xAI first, then Sol/Terra/Luna; each row shows provider suffix; Grok + GPT visible together |
| partial | catalog entry missing fields | ✅ covered | Missing `name` → fall back to model id string; missing `provider` parse → default `xai` (label shows `(xAI)` for new ship defaults always set); missing description → empty secondary line |
| overflow | list-collection / nav dropdown | ✅ covered | Existing picker: vertical scroll within modal/dropdown; label `truncate_str` to remaining columns; description wraps within row width budget |
| zero-one-many | model list-collection | ✅ covered | Zero → empty copy; one → single row, still with provider suffix; many → scrollable flat list, order stable, no grouping headers |
| long-text | row label / description | ✅ covered | Long names truncate with existing truncation; descriptions wrap then clip at area height; provider suffix is short (` (xAI)` / ` (Codex)`) so it should remain visible when truncation is end-ellipsis — **prefer truncating the family name before dropping the provider suffix** when implementable; if reuse-only, full-string truncate is acceptable with test note |
