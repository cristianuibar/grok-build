## Spacing Scale

Declared values for **terminal layout** (cell-based; multiples of 4 where width budgets apply):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 1 col | Gap before provider badge space / between glyph and label |
| sm | 2 cols | Indent steps, gap before right-label, trailing pad before border |
| md | 4 cols | Section gutter inside modal/popup frames |
| lg | 8 cols | Modal horizontal padding (existing popup frame) |
| xl | 16 cols | Reserved ‚Äî do not introduce new wide gutters this phase |
| 2xl | 24 cols | Reserved |
| 3xl | 32 cols | Reserved |

**Row geometry (locked to existing picker):**

| Element | Contract |
|---------|----------|
| List row height | 1 cell primary line; optional +N wrapped description lines |
| Fold/prefix width | 2 cells (`‚óÜ ` / `‚Ä∫ `) |
| Trailing pad | 1 cell before scrollbar/border |
| Right-label gap | 2 cells when `right_label` non-empty |
| Indent step | 2 cells per indent level |

**Exceptions:**

- Icon-only touch targets (44px) ‚Äî **N/A** (keyboard/mouse terminal, not mobile web)
- Do not invent CSS rem/px spacing outside the cell table above
- **Character-cell chrome (xs=1 / sm=2, fold/prefix 2 cells, trailing pad 1):** locked to existing picker geometry (`views/picker.rs`). These are **not** web 4px-grid violations ‚Äî do not "fix" them to 4-col prefixes or you will break fold/glyph layout. Larger gutters (`md`‚Äì`3xl`: 4/8/16/24/32) already match the standard set.

## Typography

Terminal monospaced ‚Äî ‚Äúsize‚Äù is **role weight + Theme color**, not px. Exactly **3 roles**, **2 weights**.

| Role | Size (cells) | Weight | Line Height | Theme / style |
|------|--------------|--------|-------------|---------------|
| Body (primary row label) | 1 row | regular (400) / **bold when selected** | 1.0 (single cell row) | `text_primary`; selected + `Modifier::BOLD` |
| Label (provider suffix, meta, CLI bullets) | 1 row | regular (400) | 1.0 | Provider suffix part of label string; CLI `* `/`- ` prefix dim |
| Secondary (description under row) | 1+ wrapped rows | regular (400) | 1.0 | `gray` / `text_secondary` via picker meta/description styles |
| Display | ‚Äî | ‚Äî | ‚Äî | **not used this phase** (no titles beyond existing modal chrome) |

**Rules:**

- Max **2** font weights: regular + bold (selected row only)
- No italic for model names; italic reserved for existing ghost-text elsewhere
- Fuzzy match highlight uses `theme.fuzzy_accent` on matched spans only
- Prompt chrome model name continues to use `theme.accent_model` (teal on GrokNight) ‚Äî do not recolor per provider this phase

## Color

60/30/10 against **GrokNight** defaults (other themes inherit role mapping, not hard hex):

| Role | Token (GrokNight) | Approx | Usage |
|------|-------------------|--------|-------|
| Dominant (60%) | `bg_base` / terminal bg | `#141414` | List/modal background, empty space |
| Secondary (30%) | `bg_highlight` / `bg_hover` / `bg_visual` | `#242424` / `#2c2c2c` | Selected row, hover row, popup panels |
| Accent (10%) | `accent_model` + `fuzzy_accent` | teal `#1abc9c` / blue `#7aa2f7` | **Only** prompt model name + fuzzy match highlights |
| Destructive | `accent_error` | `#f7768e` | Not used for catalog rows this phase; reserved for future gate/errors |

**Accent reserved for (explicit ‚Äî never ‚Äúall interactive elements‚Äù):**

1. Current model name in the prompt status / info line (`accent_model`)
2. Fuzzy-match character highlights inside the model picker search (`fuzzy_accent`)

**Provider label coloring:** **none this phase.** Provider is plain text in the name string (` (xAI)` / ` (Codex)`). Do not introduce per-provider accent colors until a later polish phase.

**Text hierarchy:**

| Content | Color |
|---------|-------|
| Unselected model name | `text_primary` |
| Selected model name | `text_primary` + bold + selected row bg |
| Description | `gray` / secondary |
| Dimmed / non-selectable (if any) | `gray_dim` |
| CLI current marker `*` | default stdout; no ANSI theme required for v1 list |

## Interaction Contract

### List structure (MOD-01, MOD-02)

| Rule | Contract |
|------|----------|
| Shape | **Single flat mixed list** ‚Äî no tabs, no provider filter mode, no session-global provider scope |
| Order | **xAI/Grok entries first** (catalog insertion order), then GPT-5.6 **Sol ‚Üí Terra ‚Üí Luna** |
| Binding | Every row‚Äôs underlying entry has explicit `provider`: `xai` \| `codex` (for Phase 4; visible via name suffix now) |
| Visibility | GPT rows **always listed** even if Codex credentials missing |
| Default model | Remain **`grok-build`** (`provider: xai`) |
| Scope this phase | Catalog + labels + visibility only ‚Äî **no** Codex routing, **no** missing-provider block modal |

### Surfaces

| Surface | Behavior |
|---------|----------|
| `/model` / `/m` | Arg dropdown lists all user-selectable models in catalog order; display = name (with provider suffix); description = catalog description; resolve by name **or** id |
| Settings ‚Üí Default model | DynamicEnum from active catalog; same names/descriptions; no provider filter |
| `bum models` | Prints default id + flat list with id and provider-bearing name; mark current with `*` |
| ACP `ModelInfo` | `name` already includes provider suffix; optional meta key `provider` (`"xai"` \| `"codex"`) for machine consumers ‚Äî discretion allowed if wire already crowded; UI must not depend on meta alone for the visible label |

### Selection chrome

| State | Visual |
|-------|--------|
| Unselected row | primary text, base bg |
| Selected / keyboard focus | bold primary + `bg_visual` (or embedded modal selected style) |
| Mouse hover | `bg_hover` |
| Current model in slash list | name + ` (current)` in display string |
| Search / filter | existing fuzzy filter; no matches ‚Üí empty list with empty-state copy (no crash) |

### Explicit non-goals (defer)

- Login prompt when picking Codex without auth ‚Üí Phase 6  
- Per-provider color badges / icons ‚Üí later polish  
- Capability matrix / effort matrix UI ‚Üí MOD-V2-01  
- BYOK custom GPT rows as product requirement ‚Üí deferred
