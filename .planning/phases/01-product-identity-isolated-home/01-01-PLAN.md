## Phase Goal

**As a** bum user, **I want to** have all product state under isolated `~/.bum` (or `BUM_HOME`), **so that** stock Grok/Codex homes are never used as bum's product root.

<objective>
Cut over the single source of truth for product home in `xai-grok-config` and path display labels so default is `~/.bum`, override is `BUM_HOME` only, managed binary leaf is `bum`, and path UI labels match (per D-HOME, D-WRITERS, D-SCOPE path-label exception). Addresses review concern: OnceLock multi-scenario tests must be process-isolated or pure-helper-tested.

Purpose: Almost all product-state writers hang off `grok_home()`; fixing SoT + display is the foundation for isolation (ID-03).
Output: Pure resolver + production OnceLock, mandatory lib docs, display helpers, process-isolated binary tests proving `.bum` / `BUM_HOME` / ignore `GROK_HOME`.
</objective>

|----------|-------------|
| Process env → product home | Untrusted env (`BUM_HOME`, leftover `GROK_HOME`) selects credential/session root |
| Product home → disk writers | Auth tokens, sessions, MCP creds written under resolved home |
