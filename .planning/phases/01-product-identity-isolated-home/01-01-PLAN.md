**As a** bum user, **I want to** have all product state under isolated `~/.bum` (or `BUM_HOME`), **so that** stock Grok/Codex homes are never used as bum's product root.

Purpose: Almost all product-state writers hang off `grok_home()`; fixing SoT + display is the foundation for isolation (ID-03).
Output: Updated `paths.rs` (production + in-module unit tests in one green step), display helpers, and isolated-binary tests proving `.bum` / `BUM_HOME` / ignore `GROK_HOME`.
</objective>
