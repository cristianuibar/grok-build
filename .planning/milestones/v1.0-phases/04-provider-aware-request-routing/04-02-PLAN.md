## Production authority contract (review HIGH — locked)

`resolve_provider_route` is **not** test-only. Plans 03–04 **must** call it when choosing provider-default base_url:

| Consumer | Must call resolve_provider_route for |
|----------|--------------------------------------|
| `default_models()` stamping (Plan 03) | provider-default `base_url` (when no JSON override) |
| Provider rebind after `ConfigModelOverride` (Plan 03) | re-normalize base when provider changes and no explicit base_url override |
| `resolve_credentials*` when base empty / session oauth gate (Plan 03) | `session_oauth_allowed` + default base fill |
| prepare / reconstruct policy helpers (Plan 04) | slot + oauth-allowed decisions when needed |

Do **not** re-implement parallel if/else base tables in those call sites.

## First-party host / OAuth leak policy (review HIGH — locked)

When a model has an **explicit non-empty base_url override** that is **not** a first-party host for that provider:

- `ProviderRoute.base_url` = the override (D-04 wins for URL)
- `credential_slot` still follows `ModelProvider` (identity of which slot *could* apply)
- `session_oauth_allowed: false` — production credential resolve **must not** attach provider session OAuth bearer to that host
- Sampling for that model requires **own/model credential** (`own_credential` / entry api_key/env_key) or fails closed at sample time (Plan 05)

First-party hosts (allowlist, pure helpers):

- **xAI / proxy:** existing first-party xAI / cli-chat-proxy classification already used by `is_first_party_xai_url` / proxy detection — reuse; do not invent a second table if one exists
- **Codex:** host/path matching `chatgpt.com` Codex backend (and configured `endpoints.resolve_codex_base_url()` host). Not `api.openai.com` Platform as product first-party for OAuth slot (D-15)

When override is empty: provider default base is used; `session_oauth_allowed` is still derived from whether the **final** `base_url` is first-party for that provider (true for stock defaults). Empty override does **not** force `session_oauth_allowed: true` independently of host classification.

**Cycle-3 MEDIUM (locked):** `provider=Xai` + non-empty custom `models_base_url` / override to a non–first-party host (e.g. `https://byok.example/v1`) → `session_oauth_allowed: false` (same rule as Codex custom host). Do not allow session OAuth solely because provider is Xai when the final URL is custom.

## codex_base_url field shape (review LOW — intentional)

Use `String` matching `xai_api_base_url` (not `Option<String>` like proxy). Document: blank → default via `resolve_codex_base_url`. Asymmetry with proxy is intentional and matches xAI API base field style.
