## Phase Goal

**As a** bum user who switches models mid-session, **I want** the next sample to use the new model’s backend URL and credential slot, **so that** Grok turns stay on xAI and GPT turns use Codex credentials without process restart.

<objective>
Wire pure route + dual-key credentials into production sampling builders (per D-03, D-08, D-09, D-10, D-11): `prepare_sampling_config_for_model`, `ModelsManager::sampling_config`, `session/.../model_switch.rs`, `session/commands.rs` SetSessionModel carrier, and `reconstruct_full_config` BearerResolver policy. Purpose: Complete next-sample routing seam Phase 5/6 will call.

</objective>

## Shared production cores + test hooks (cycle 1+2 HIGH — locked)

Integration tests cannot call `pub(crate)` / `pub(super)`. Extract **public** pure/shared functions that production methods **must** call (thin wrappers). Tests call **those same functions** — never a parallel field-copy that can drift from model_switch / reconstruct.

### Required public surface (names may vary slightly; behavior fixed)

1. `pub fn session_key_for_model_provider(provider: ModelProvider, xai, codex) -> Option<&str>`
2. `pub fn should_attach_xai_auth_manager_bearer_resolver(provider: Option<ModelProvider>, session_token_gate_active: bool) -> bool` — true **only** for `Some(ModelProvider::Xai) && gate`; false for `None`, `Some(Codex)`, gate inactive
3. `pub fn select_provider_access_token(store: &AuthStore) -> Option<GrokAuth>` (or key String) — selection rules below
4. `pub fn prepare_sampling_credentials(model, endpoints, xai_session, codex_session, ...) -> ResolvedCredentials` — body used by prepare + ModelsManager (pass EndpointsConfig per Plan 03)
5. **`PreparedSamplingConfig`** (or equivalent named type) carrying at minimum:
   - `sampler_config: SamplerConfig` (or the fields SetSessionModel already needs)
   - `auth_type: CredentialsAuthType` / chat-state auth type enum already used by `update_credentials`
   - `provider: ModelProvider` (known models only — prepare fails closed or refuses unknown before switch if model not in catalog)
6. **Production transform A** — `pub fn apply_prepared_sampling_to_chat_state_fields(prepared: &PreparedSamplingConfig, existing_credentials, context_window, ...) -> (chat_state SamplingConfig, Credentials)`  
   - Maps prepared sampler fields + **prepared.auth_type** + prepared api_key into the exact chat-state structs that `model_switch.rs` writes today via `update_sampling_config` / `update_credentials`
   - **Production `session/.../model_switch.rs` MUST call this** (or inline call the same function body). Forbidden: handler re-deriving `auth_type` via `resolve_chat_state_auth_type(model, xAI AuthManager key, …)`
7. **Production transform B** — `pub fn reconstruct_attach_policy_from_facts(facts: &ModelAuthFacts, session_token_gate_active: bool) -> bool` (or full `pub fn reconstruct_sampler_config_from_chat_state(chat sampling + credentials + facts + gate) -> ReconstructDecision { base_url, api_key, attach_xai_bearer }`)  
   - **Production `reconstruct_full_config` MUST use** `should_attach` / this transform for bearer_resolver wiring
8. Extend `ModelAuthFacts` with `pub provider: Option<ModelProvider>`:
   - Catalog hit → `Some(entry.provider)`
   - Empty model id / config unavailable / model absent → **`None`** (do **not** use `ModelProvider::default()` / Xai)
9. `pub fn read_provider_auth_store(path, provider) -> Result<Option<AuthStore>, AuthStoreReadError>` (or `io::Result` + typed kind) — see store section

### Switch test contract (cycle 2 HIGH — locked)

**Forbidden:** a `next_sample_after_model_switch` helper that **reimplements** field copies differently from production.

**Required:** `switch_changes_next_sample_route` composes, in order:

1. Build `PreparedSamplingConfig` for model A via the same prepare credential path production uses (public prepare_sampling_credentials / sampling fields)
2. Call **transform A** → chat-state SamplingConfig + Credentials
3. Build facts for model B (catalog Option provider)
4. Call **transform B** / should_attach with those facts
5. Assert base_url, api_key slot, and attach_xai_bearer differ correctly when switching xAI → Codex (and reverse if useful)

Optional thin `next_sample_after_model_switch` **may exist only as a composition of A+B** with zero extra field mapping logic.

Optional `#[doc(hidden)]` module ok if re-exported for integration tests.

## auth_type / provider carrier (cycle 2 HIGH — locked)

Today `SetSessionModel` carries only `SamplerConfig` (`session/commands.rs`). The session handler then re-resolves `auth_type` with the xAI AuthManager key (`model_switch.rs`). That is the xAI bias defect.

**Required data-flow change (pick one; document choice in SUMMARY):**

| Option | Change |
|--------|--------|
| **A (preferred)** | Add fields on `SessionCommand::SetSessionModel`: `auth_type` (chat-state type) and optionally `provider: ModelProvider`. Handler `prepare` path fills them from `PreparedSamplingConfig`. model_switch writes them **verbatim** into `update_credentials` — no `resolve_chat_state_auth_type` with xAI session_key. |
| **B** | Replace/augment payload with `PreparedSamplingConfig { sampler_config, auth_type, provider }` consumed by model_switch. |

Either way:

- handlers/model_switch.rs builds the carrier from prepare output
- session model_switch **must not** call `resolve_chat_state_auth_type` with AuthManager xAI key for the new model
- api_key remains the prepared provider-correct key (already on SamplerConfig)

## Provider store load (cycle 2 MEDIUM — locked)

`read_provider_auth_store` / disk load for Codex fixture selection:

- Return type **`Result<Option<AuthStore>, E>`** (or enum with Missing / Parse / UnsupportedVersion / Io), **not** bare `Option` that collapses all failures to None
- Missing file → `Ok(None)` (or `Err` with Missing variant that callers treat as no credentials)
- Malformed JSON / unsupported version / I/O → `Err(...)` with **redacted** diagnostic (path ok; no tokens) via tracing warn
- Credential resolve still **fail closed** (no key) on any Err / empty slot — distinguish only for diagnostics and tests
- Test: parse-failure path is distinguishable from missing-file path (unit or provider_routing)

## Codex token selection (review HIGH — locked)

**Forbidden:** “first entry in BTreeMap” / `values().next()`.

**Required** `select_provider_access_token` (mirror `lookup_auth` quality):

1. Consider each scope in the provider’s `AuthStore`
2. Skip `AuthMode::WebLogin` (same as `lookup_auth`)
3. Skip blank/whitespace keys
4. Prefer non-expired tokens (use existing `is_expired` / buffer helpers when available at this layer)
5. Prefer `AuthMode::Oidc` (session OAuth) over `ApiKey` over `External`
6. If multiple remain, prefer the scope key matching any future Phase 5 constant when present; else stable preference: Oidc winners sorted by scope string for determinism (document as fixture-stable, not “first arbitrary”)
7. Return None if nothing valid

Phase 5 will pin the real ChatGPT OAuth scope constant; selection function must accept that without redesign.

## Provider recovery (cycle 1+2 HIGH — locked)

- **Do not** decide bearer attach from URL host heuristics as primary.
- Use `ModelAuthFacts.provider: Option<ModelProvider>` from catalog lookup of `cfg.model`.
- Catalog hit → `Some(provider)`. Empty model / config unavailable / absent model → **`None`**.
- **Never** fill missing catalog with `ModelProvider::Xai` / `Default` for attach policy.
- Attach xAI live resolver **only** when `provider == Some(ModelProvider::Xai)` and session-token gate active.
- URL may still drive `inject_url_derived_headers` and SessionTokenAuthGate host checks as today; **credential ownership / resolver attach uses Option provider field**.
