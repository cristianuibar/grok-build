**Discretion locks this plan:**
- Status module: new `auth/status.rs` (pure; no CLI I/O) — CLI wire in Plan 04
- Mutation result may generalize `XaiStoreMutation` → `ProviderStoreMutation`
- `mutate_xai_store_or_prune` may delegate to generic provider API with AuthProvider::Xai
- **Public API (review):** export intentional public provider-storage API for integration tests (path + AuthProvider). Do not leave only `pub(crate)` while lifecycle integration tests call mutate/clear.
- **AuthProvider enum** `{ Xai, Codex }` with `as_str()` → `"xai"` / `"codex"`; mutation APIs take enum (or parse allow-list only)
- **usable semantics (review):**
  - `logged_in`: persisted OAuth session/token record exists (nonblank access key and/or refresh_token / session entry)
  - `usable`: hard-unexpired access token **or** (refreshable OAuth session: refresh_token present and not marked permanently invalid)
  - Must **not** derive usable solely from `select_provider_access_token(...).is_some()` (that helper returns expired tokens when no fresh candidate exists)
- plan field: display em-dash unless schema intentionally extended (no dedicated plan field required)
