- Product label is **`bum`** only (not "Build Using Multiagents" / "bum (CLI)")
- Text output shape is **`bum {version}`** — same layout as stock `grok {version}`, only rename the product token
- Keep **`display_version_with_commit`** + channel label plumbing so version/hash stay present
- JSON `--json` payload stays **`currentVersion` + `channel` only** — no product field required for ID-02

### Fix scope
- Primary fix is the **`Command::Version` human-readable println** in pager-bin `main.rs` (the known live gap)
- Do **not** open a full residual-string rebrand pass; only version path + regression gate
- Crate/package names (`xai-grok-*`) stay as-is — product identity is the binary + user-facing strings
- Incidental user-facing `grok` strings outside the version command are **out of scope** unless they are on the same version print path

### Regression gate
- Add a **unit/integration test** that the version text product token is `bum` (not `grok`)
- Prefer a **hermetic binary probe** (spawn `bum version` under isolated home) asserting stdout starts with `bum ` / does not start with `grok `
- Add **scoped static grep** against the version path hard-coding regression (`"grok {}"` / equivalent on the version branch)
- Dual-login live UAT **not required** for this gap close

### Claude's Discretion
- Exact constant vs inline `"bum"` for the product token (match existing Phase 8 patterns like `UPDATE_STATUS_PRODUCT_NAME` / `MINIMAL_WELCOME_PRODUCT_NAME` if cheap)
- Whether to extract a tiny pure formatter helper for unit testing vs assert via binary spawn only
