## Summary

Milestone audit (`v1.0-MILESTONE-AUDIT.md`) marks v1.0 `gaps_found` solely because `bum version` still prints stock product identity. Live HEAD probe confirms:

```text
$ target/debug/bum version
grok 0.1.220-alpha.4 (f3eccc6)

$ target/debug/bum --version
bum 0.1.220-alpha.4 (f3eccc6)

$ target/debug/bum version --json
{"currentVersion":"0.1.220-alpha.4 (f3eccc6)","channel":"unknown"}
```

Root cause is **not** missing rebrand infrastructure. Phase 8 already branded clap `name = "bum"`, welcome/hero, OAuth return, minimal welcome, update status product name, and composition-root banner/crash/serve lines. The **version subcommand** human-readable branch was never retargeted: it still formats with a hard-coded `"grok {}"` prefix while correctly calling `xai_grok_version::display_version_with_commit` for version/hash/channel. [VERIFIED: codebase + live binary probe]

Clap’s top-level `--version` / `-V` / `-v` path already prints `bum …` because clap uses `name = "bum"` + `version_with_channel()`. JSON already omits product name (locked). This phase is a **surgical one-site product-token rename + regression gate**, not a stack or architecture change.

**Primary recommendation:** Replace `"grok {}"` with a Phase-8-style product constant (or pure `format_version_cmd_line`) printing `bum {display_version_with_commit(...)}`; add a pure unit test beside existing `p8_bin_*` helpers, a hermetic `CARGO_BIN_EXE_bum version` stdout assertion, and a scoped static grep forbidding `"grok {}"` on the version path.

### Anti-Patterns to Avoid

- **Full residual rebrand pass:** CONTEXT defers non-version `grok` strings; expands phase and reopens C1-H1 inventory.

## Common Pitfalls

### Pitfall 1: Fixing the wrong version surface
**What goes wrong:** Engineer “verifies” with `bum --version` (already `bum …`) and ships without fixing `bum version`.  
**Why it happens:** Two distinct code paths (clap flag vs `Command::Version`).  
**How to avoid:** Gate on the **subcommand** path only; acceptance command is `bum version` (and hermetic equivalent).  
**Warning signs:** Live audit evidence still shows `grok …` after “green” clap tests.

### Pitfall 2: Unit-only green, binary still stock
**What goes wrong:** Helper says `bum` but match arm still `println!("grok {}", …)`.  
**Why it happens:** Incomplete wire-up.  
**How to avoid:** Locked dual gate: pure unit **and** hermetic binary stdout.  
**Warning signs:** `p8_bin_*` pass but `cargo test -p xai-grok-pager-bin --test home_isolation` (or dedicated probe) fails.

### Pitfall 3: Scope creep into residual `grok` strings
**What goes wrong:** Plan expands to wrap_cmd / spawn allow-list / internal client ids.  
**Why it happens:** Broad `rg grok` after noticing audit residual debt.  
**How to avoid:** CONTEXT deferred list; only version path + its regression greps.  
**Warning signs:** Diff touches files outside `xai-grok-pager-bin` for ID-02.

### Pitfall 4: Breaking channel/hash display
**What goes wrong:** Replace entire line with a static `"bum"` or drop `display_version_with_commit`.  
**Why it happens:** Over-eager string replace.  
**How to avoid:** Keep call signature; only change the prefix token. Expected shape: `bum 0.1.220-alpha.4 (f3eccc6)` optionally with channel label suffix.  
**Warning signs:** Output lacks `(` short hash `)` or loses channel label when pointer is present.

### Pitfall 5: Formatter / test blast radius
**What goes wrong:** `cargo fmt --all` reflows unrelated files; full workspace tests timeout.  
**Why it happens:** Prior milestone pain (STATE.md / Phase 10 notes).  
**How to avoid:** `cargo fmt -- crates/codegen/xai-grok-pager-bin/src/main.rs …` check-only; tests scoped to package + filters.  
**Warning signs:** Huge unrelated diff in `git status`.

### Pitfall 6: Vacuous static grep
**What goes wrong:** Grep path so narrow it always passes, or so wide it fails on deferred strings.  
**Why it happens:** Poor ownership of residual gate.  
**How to avoid:** Exact stock format `"grok {}"` under `xai-grok-pager-bin` (pre-fix: must match once; post-fix: zero).  
**Warning signs:** Gate green before the product fix lands.
