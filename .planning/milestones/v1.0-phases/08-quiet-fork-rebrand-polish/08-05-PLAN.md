## Discretion (document in SUMMARY)

1. **Remote policy (mandatory for quiet fork / C1-H3):** remote settings for `telemetry_mode` / `telemetry_enabled` and `feedback_enabled` are **restrictive-only** — remote may further disable or stay off; remote-true + local-unset must resolve Disabled/false. Explicit env (`GROK_TELEMETRY_ENABLED`, `GROK_FEEDBACK_ENABLED`) and explicit local/admin config/requirements still may enable for developers.
2. **Sentry:** Force Config.disabled = true at pager-bin init for quiet fork hard privacy (still leave crate linked). Do not advertise opt-in UX.
3. **Feedback UX:** Discoverable command stays; short-circuit at enter; do not open collect mode; no Effect::SendFeedback.
4. **OTLP (C1-H2):** Prefer default `InstrumentationMode` not Server when env unset, and `OtelExporterConfig.enabled` false via `build_default_otel_layer_config` when telemetry is not explicitly enabled — both TUI and non-TUI `init_tracing*` / `otel_layer` paths inherit. Local Log/Chrome instrumentation remains allowed under D-11.
