# Phase 8 — UI Design Contract

> Visual and interaction contract for **quiet fork & rebrand polish**
> (ID-02, OPS-01, OPS-02).
> Product is a **ratatui terminal TUI + CLI + OAuth browser HTML**, not a web
> app. This phase is **copy/chrome rebrand + disabled-update/feedback messaging**
> only — **not** a new design system, color palette, layout, or theme.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Typography

Terminal monospaced — roles map to **Theme + Modifier**, not px. Exactly **3 roles**, **2 weights**. **No new display type.**

| Role | Size (cells) | Weight | Line Height | Theme / style |
|------|--------------|--------|-------------|---------------|
| Body (hero product name in Full/HeroInline badges, project picker question, system blocks, CLI lines) | 1 row | regular (400); **bold** only where stock already bolds product badge | 1.0 | `text_primary` (+ `Modifier::BOLD` on product badge spans that already use it) |
| Label (menu keys, option labels, field keys) | 1 row | regular (400); bold on selected rows only (existing) | 1.0 | existing menu / QuestionView styles |
| Secondary (version number, channel, hero subtitle, dim hints, gray footer badge bits) | 1+ rows | regular (400) | 1.0 | `gray` / `text_secondary` / existing |
| Display | — | — | — | **not used** this phase |

**Rules:**

- Max **2** font weights: regular + bold (same as stock hero badge)
- Rebrand is **string substitution**, not new type roles
- Model names in catalog/prompt continue Phase 3 `accent_model` path — **unchanged**
- **Never** style or print access tokens, refresh tokens, JWTs, or `Authorization` headers

## Color

**No color redesign.** 60/30/10 against **GrokNight** defaults (other themes inherit roles):

| Role | Token (GrokNight) | Approx | Usage this phase |
|------|-------------------|--------|------------------|
| Dominant (60%) | `bg_base` / terminal bg | `#141414` | Welcome background, empty space |
| Secondary (30%) | `bg_highlight` / `bg_hover` / `bg_visual` | `#242424` / `#2c2c2c` / `#363636` | Existing panels/selection only |
| Accent (10%) | `accent_model` + `fuzzy_accent` | teal / blue | **Unchanged** — model name + fuzzy match only |
| Destructive / fail | `accent_error` | `#f7768e` | Only if an error-styled path already exists; disabled update/feedback use **system/primary or secondary** informational tone (not alarm red) |

**Accent reserved for (explicit — never “all interactive elements”):**

1. Current model name in prompt status (`accent_model`) — unchanged
2. Fuzzy-match highlights in pickers (`fuzzy_accent`) — unchanged
3. Existing user/path accents on workspace trust path (`accent_user`) — unchanged
4. **Not** for product name “bum”, update-disabled lines, or feedback-disabled lines (those are Body/Secondary text)

**Provider coloring:** none. Labels stay plain `xAI` / `Codex`.

**Rebrand does not introduce** a new product brand color, logo recolor, or “bum theme.”

## Product vs model brand (critical)

| Kind | User-visible rule | Examples |
|------|-------------------|----------|
| **Product shell** | Always **bum** | CLI name, clap about, hero badge, project picker, OAuth return, feedback/update messaging, workspace warning subject, ZDR blocked product line, product-named subscribe/credits copy, version banners |
| **Model / catalog brands** | **Keep** | Catalog `name` for id `grok-build` → **Grok Build** / **Grok Build (xAI)**; other Grok / GPT-5.6 / Sol / Terra / Luna display names; model id slugs |
| **Provider commercial products** | **Keep** | SuperGrok, ChatGPT, Codex as external product names in billing/upsell where they refer to xAI/OpenAI offerings |
| **Internal identifiers** | **Keep** | `xai-grok-*` crates, `GROK_*` env (non-home), wire paths, telemetry event type names in code |

**Half-rename ban:** User-facing chrome must not still read as “Grok Build” *the product*. Model rows may still say “Grok Build (xAI)” because that is the **model** label.

### Feedback (quiet fork — OPS-02 adjacent)

**UX decision (CONTEXT discretion):** Keep `/feedback` **discoverable** but **disabled** with a clear local message. No HTTP POST / product-event phone-home for feedback by default.

| Element | Locked copy |
|---------|-------------|
| Primary path when user runs `/feedback` or submits feedback mode | System/toast: `Feedback is disabled in bum (no phone-home).` |
| Optional one-line hint (if second line needed) | `Local logs under ~/.bum remain available for debugging.` |
| Empty submit (if mode still opens) | `Please provide feedback text.` → **prefer not entering collect mode**; if mode is blocked at command entry, only the disabled message fires |
| Success thank-you (stock) | **Remove** `Thanks for the feedback! The Grok Build team is on it.` — never show team phone-home thank-you |
| Error `Couldn't send feedback: …` / `couldn't send feedback: …` | Must not appear on the quiet default path (no send attempt). If a residual network path is hit in misconfig, map to the same disabled message rather than xAI team framing |

**Interaction:**

1. User invokes `/feedback` (or feedback keybinding if any) → **immediate** system block / toast with disabled copy
2. **No** `Effect::SendFeedback` / Mixpanel / Sentry user-feedback emission on default quiet fork
3. Prompt does not stay in feedback input mode after the disabled message (return to Normal)

### Auto-update / `bum update` (OPS-01)

**Behavior:** stock xAI auto-update channel **hard-off**. Explicit update path is **no-op + clear message**.

| Element | Locked copy |
|---------|-------------|
| Explicit `bum update` (and equivalent Ctrl+U / in-app update action if present) | `Stock auto-update is disabled in bum. Install or build locally to upgrade.` |
| Optional second line (CLI only, if helpful) | `This fork does not download updates from the stock x.ai channel.` |
| Startup background update check | **Silent skip** — no toast, no banner, no “new version of Grok Build” prompt |
| Forced / min-version stock policy | **Ignore** — never block the UI on stock Grok Build upgrade |
| Stock strings to **eliminate** from user path | `A new version of Grok Build is available…`, `Grok Build - v…` as product identity in update printers (use `bum` if any status line remains for local version display) |
| Local version display only (no network) | `bum - v{current_version}` acceptable; must not imply stock channel is live |

**Settings:** If `auto_update` setting remains visible, default **off/false** and user-facing description must not promise stock channel updates. Prefer description: `Stock update channel (disabled in bum)`.

| Model catalog names | **Do not** rename “Grok Build” model display |
