**Research-critical:** Do not full-union all bundled defaults after prefetch — only `provider=codex` rows. Preserve enterprise `has_custom_endpoint()` skip of built-ins.

**Collision policy (review cycle 2):** Bundled Codex defaults are authoritative for their three wire ids **and** for final list position among themselves. After prefetch replace, when `!has_custom_endpoint`:

1. Collect every `default_model_entries` row with `provider == Codex` (the three ship ids, JSON order Sol→Terra→Luna).
2. **`shift_remove` (or equivalent) every one of those keys from `resolved`** if present — discard remote provider/name and any mid-list position.
3. **Append** all three bundled entries in that JSON order onto the end of `resolved`.

Result: remote/xAI rows first (minus stripped gpt keys), then Sol→Terra→Luna even if prefetched listed Terra first or only Terra. Do **not** use IndexMap replace-in-place for colliding keys (that preserves a bad index and can put Terra before remote Grok).

**Q1 RESOLVED:** Empty `Some(IndexMap::new())` still gets Codex re-append when `!has_custom_endpoint`. Encode as `empty_prefetch_still_gets_codex_defaults`.

**Harness:** All verifies use `cargo test -p xai-grok-shell --test model_catalog …`. One TESTNAME filter per cargo invocation or full binary. No bare `| tail`. Do not use shell `--lib`. Do not rewrite remote xAI names for non-Codex keys (Q2 RESOLVED — Plan 01 leaves remote names as returned for ordinary Grok remotes).

**File ownership note:** Only `config.rs` (merge logic) and `tests/model_catalog.rs` are modified. `models.rs` is read-only inspection for `available_models` unless a bug forces a one-line fix (prefer zero change).
